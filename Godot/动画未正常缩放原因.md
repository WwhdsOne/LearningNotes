总结一下，之前缩放比例 `scale.x` 看似没生效的**根本原因**是：

**你的脚本在 `_process()` 里每帧都直接给 `scale.x` 赋值为 `1` 或 `-1`，把 `_ready()` 中设置的基础放大倍数（25.0）覆盖掉了。**

- 无论角色移动还是待机，`update_movement_animation()` 和 `update_idle_animation()` 都会执行 `scale.x = 1` 或 `scale.x = -1`，导致 `scale.x` 在运行时永远是 1 或 -1，而不是 25.0。
- 视觉上 X 方向就被压回了原始宽度的 1 倍（因为精灵图本身可能很窄，变成 1 个像素宽几乎看不见），所以你觉得“第一个参数没生效”、“移动不了”。
- 你在编辑器里手动把 `scale.x` 改成 5 后保存又变回 1，也是因为运行游戏时脚本的 `_process` 立刻又把它设成了 1（或在编辑器中如果脚本带有 `tool` 关键字，也会在编辑器里执行）。

正确的做法是**不要用直接赋值 `scale.x` 来控制翻转**，改用 `flip_h` 属性，这样基础缩放就能保持不变，翻转也正常工作。

`flip_h` 是 Godot 中 `Node2D` 及其子节点（比如 `Sprite2D`、`AnimatedSprite2D`）的一个内置布尔属性，专门用来**水平翻转节点**。

---

### 它的作用

- 当 `flip_h = true` 时，节点会在 **X 轴方向镜像显示**，也就是左右翻转。
- 翻转不会改变节点的 `scale`、`position`、`offset` 等属性的实际值，只是改变渲染结果。
- 如果你需要垂直翻转，可以用 `flip_v`。

---

### 基本用法

```gdscript
# 让角色面向右边（假设原始资源是朝左画的）
$AnimatedSprite2D.flip_h = true

# 让角色面向左边
$AnimatedSprite2D.flip_h = false
```

也可以直接在编辑器里勾选 `Sprite2D` 或 `AnimatedSprite2D` 检查器中的 `Flip H`。

---

### 它和 `scale.x` 翻转的区别

很多人最早会用 `scale.x = -1` 来实现水平翻转，但这个方法有几个缺点，而 `flip_h` 完全避免了它们：

| 方法 | `scale.x = -1` | `flip_h` |
|------|----------------|----------|
| **影响缩放** | 会影响节点的整体缩放值，如果你本来把角色放大了 5 倍，`scale.x = -1` 之后会变回 1 倍宽，破坏放大效果。 | 完全不影响 `scale`，你设置的放大倍数会保留。 |
| **物理碰撞** | 可能会反转 `CollisionShape2D` 或 `CollisionPolygon2D` 的形状方向（取决于是否作为子节点受 `scale` 影响）。 | 仅影响视觉渲染，物理碰撞形状保持原样。 |
| **脚本复杂度** | 需要记住基础缩放值，每次翻转时做乘法：`scale.x = base_scale * -1`，容易出错。 | 一条赋值语句，清晰明了。 |
| **与其他变换的叠加** | 如果你同时还旋转了节点，`scale.x = -1` 可能产生难以预料的结果。 | 翻转总是按局部 X 轴镜像，行为稳定。 |

简单来说，**凡是只想让图片左右镜像显示的，都应该用 `flip_h`**，而不是去改 `scale.x`。这样缩放、旋转、动画都能正常工作，不会互相干扰。
