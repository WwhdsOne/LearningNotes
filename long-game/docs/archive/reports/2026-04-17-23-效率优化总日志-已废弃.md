# 效率优化总日志（2026-04-17 至 2026-04-23）

## 概述

本日志汇总了从 4 月 17 日至 4 月 23 日期间，各分支涉及效率优化的核心提交与改动。优化工作主要围绕 **实时链路重构**、**框架迁移**、**载荷瘦身** 和 **序列化升级** 四个方向展开。

---

## 一、实时状态链路与点击性能优化

**提交**: `1387824` (4月21日)  
**分支**: main  
**报告**: [2026-04-21-realtime-optimization-report.md](./2026-04-21-realtime-optimization-report.md)

### 核心改动

| 改动点 | 说明 |
|--------|------|
| 事件驱动 | 引入 Redis 事件总线，写操作成功后发布状态变更事件，替代高频轮询 |
| SSE 拆分 | 将完整状态推送拆为 `public_state`（公共态）和 `user_state`（个人态） |
| Redis 索引 | 新增 `vote:buttons:index`、`vote:equipment:index`、`vote:players:index` 索引，热路径优先走索引读取 |
| 后台分页 | `/api/admin/state` 改为摘要信息，玩家列表独立分页接口 |
| 点击链路 | 点击成功后不再回读完整状态，只返回直接相关数据 |
| Lua 缓存 | Boss 点击 Lua 脚本改用 `SCRIPT LOAD + EVALSHA`，减少脚本传输开销 |
| pprof 接入 | 新增 `/debug/pprof/` 路由，支持性能分析 |

### 效果

- 单次点击触发的状态聚合次数显著减少
- Redis 热路径从"扫描和全量读"转向"索引和定向读"
- 后台在玩家规模增长时更稳定

## 综合效果

1. **传输效率**: 公共态载荷显著压缩，按钮分页减少首屏数据量
2. **计算效率**: 事件驱动替代轮询，减少重复状态聚合
3. **存储效率**: Redis 热路径走索引，Lua 脚本缓存复用
4. **框架效率**: Hertz 原生支持更优的路由注册与请求处理
5. **序列化效率**: Sonic 替代标准库，减少 JSON 处理开销
6. **实时能力**: WebSocket + SSE 双链路保障实时通信稳定性
