# 前台三页拆分总结

## 背景

前台已经从单体 `PublicPage.vue` 拆成三个可独立维护的页面：

- `frontend/src/pages/BattlePage.vue`：战斗页，维护统计、Boss、3x3 点击区、排行、最近掉落和挂机主按钮。
- `frontend/src/pages/ProfilePage.vue`：资料页，维护登录、背包、属性、装备栏、小小英雄、强化和商店。
- `frontend/src/pages/MessagesPage.vue`：消息页，维护留言输入、留言流、公告、规则和在线玩家侧栏。
- `frontend/src/pages/PublicPage.vue`：前台壳组件，只维护顶部导航、公共头部、公告弹窗和页面切换。
- `frontend/src/pages/publicPageState.js`：前台共享状态与行为，维护登录态、实时连接、点击、资料刷新、消息加载和公共 API 调用。

## 路由边界

当前前台路由保持为：

- `/`：战斗页。
- `/profile`：资料页。
- `/messages`：消息页。
- `/admin`：后台入口，保持原状。

`PublicPage.vue` 根据路径维护 `currentPublicPage`，导航点击通过 `history.pushState` 切换前台页面。

## 数据边界

战斗页继续使用现有 `/api/buttons` 首屏数据和实时连接，实时个人增量只消费战斗必需字段：

- `userStats`
- `myBossStats`
- `combatStats` 的首屏兜底
- `recentRewards`
- `lastReward`

资料页进入时调用 `GET /api/player/profile`，获取完整资料数据：

- `inventory`
- `heroes`
- `activeHero`
- `loadout`
- `combatStats`
- `gems`
- `ownedCosmetics`
- `equippedCosmetics`
- `shopCatalog`
- `lastForgeResult`

资料页内执行装备、英雄、强化、觉醒、分解、购买、外观穿戴后，会重新刷新资料页数据。

消息页继续使用现有 `/api/messages` 和发帖接口，并展示公告与在线玩家侧栏。

## 维护规则

后续改动优先按页面落点维护：

- 战斗玩法、按钮、Boss、排行、挂机 UI：改 `BattlePage.vue`。
- 背包、装备、英雄、强化、商店 UI：改 `ProfilePage.vue`。
- 留言、公告展示、在线玩家侧栏 UI：改 `MessagesPage.vue`。
- 前台导航、页面切换、公共头部、公告弹窗：改 `PublicPage.vue`。
- 登录态、实时连接、接口调用、共享计算状态：改 `publicPageState.js`。

不要把一次性任务说明继续写进 `AGENTS.md`。
