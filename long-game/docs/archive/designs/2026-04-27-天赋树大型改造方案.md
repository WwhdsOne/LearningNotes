## 一、天赋说明文案改写

### 1.1 格式规范

每个天赋效果描述改为三要素：**干什么 → 怎么触发 → 每场可触发几次**

### 1.2 当前文案 vs 新文案（仅列举改动项）

#### 均衡攻势

| 天赋 | 旧文案 | 新文案 |
|------|--------|--------|
| 暴风连击 | 每 100 次点击触发一次追击爆发，额外造成 15 段追击总伤（单段 50%） | 每 100 次点击触发追击爆发，造成基础伤害 × 0.50 × 15 段的总追击伤害（同部位累计，触发后保留 30% 进度）。可无限触发。 |
| 蓄力返还 | 追击中断后保留 30% 蓄力进度 | 追击爆发触发后，该部位连击进度保留 30%（不归零，从 30 开始重新计数）。被动生效。 |
| 追击强化 | 追击单段倍率提升到 80% | 追击爆发的单段倍率从 50% 提升到 80%。被动生效。 |
| 连击扩展 | 追击爆发段数额外 +10 | 追击爆发段数从 15 增加到 25。被动生效。 |
| 围剿 | 每个存活部位额外增伤 12% | 每个存活部位额外增加 12% 全伤害。被动生效。 |
| 残血收割 | Boss 血量低于 25% 时，伤害倍率 2.00 倍 | 部位剩余血量低于 25% 时，伤害倍率 ×2。被动生效。 |
| 白银风暴 | 触发后连续 15 次攻击按软组织系数结算 | 任意部位被击碎时立即触发，持续 15 秒内所有部位视为软组织（×1.0 系数）。每部位击碎均可触发。 |

#### 碎盾攻坚

| 天赋 | 旧文案 | 新文案 |
|------|--------|------|
| 灭绝穿甲 | 常驻穿透 40%，100 次命中后触发 8 秒崩塌 | 常驻 40% 护甲穿透。对**重甲部位**累计 100 次命中后该部位护甲归零 8 秒（崩塌）。同一部位可多次触发。 |
| 首领猎杀 | 对首领额外伤害 +30% | 全伤害 +30%（本游戏所有敌人均为首领，与 all_damage_amplify 一致） |
| 崩塌延长 | 崩塌状态持续时间 +7 秒 | 崩塌持续时间从 8 秒延长到 15 秒。被动生效。 |
| 自动打击 | 每 20 秒自动打击一次，造成 3.0 倍伤害 | 每 20 秒自动对当前**血量最高的重甲**造成 3× 攻击力必中真伤。不需要点击。 |
| 废墟打击 | 目标处于崩塌时，额外增伤 100% | 攻击处于崩塌状态的部位时，额外增伤 100%。被动生效。 |
| 破甲转化 | 将破甲转化为增伤（转化率 50%） | 将 50% 的护甲穿透值转化为全伤害加成。被动生效。 |
| 审判日 | 每 100 次命中触发生命切割（50%） | 对**同一重甲部位**累计 100 次命中后，立即削除该部位 50% 最大生命值。**每部位每场战斗仅一次**。 |

#### 致命洞察

| 天赋 | 旧文案 | 新文案 |
|------|--------|------|
| 溢杀 | 基础暴击率 +20%，溢出暴击按 2% 转化为暴伤 | 基础暴击率 +20%。暴击率超过 100% 的部分按 2% 比例转为暴伤。弱点暴击获得 1 层死兆。 |
| 死兆共鸣 | 每层死兆额外提高 0.3% 暴击伤害 | 每层死兆叠加 0.3% 暴击伤害（例如 100 层 = +30% 暴伤）。无上限。 |
| 剥皮 | 有 30% 概率强制命中弱点，持续 5 秒 | 暴击时有 30% 概率将当前部位暂时视为弱点（×2.5 系数），持续 5 秒。 |
| 致命出血 | 附加 4 秒流血，总伤害为本次的 60% | 暴击时附加真伤 = 本次伤害 × 60%。无需持续计时，一次性结算。 |
| 斩杀预兆 | 目标血量低于 35% 时，每层死兆额外增伤 1% | 部位血量低于 35% 时，每层死兆额外 +1% 伤害（例如 47 层 = +47%）。被动生效。 |
| 死兆收割 | 消耗 30 层死兆触发一次高额收割 | 死兆达到 30 层时触发一次性必暴伤害 = 本次伤害 × 2.0。触发后死兆归零重算。 |
| 死亡狂喜 | 消耗 50 层死兆，获得 6 秒 200% 暴伤 | 死兆达到 50 层时消耗 50 层，6 秒内暴伤 +200% 且所有攻击视为弱点。可多次触发。 |
| 终末血斩 | 每 120 次暴击触发，切割目标 12% 当前生命，冷却 30 秒 | 累计 120 次暴击后削除 Boss 12% 最大生命值，30 秒冷却。 |
| 末日审判 | 叠满 2 层标记触发终结，返还 100 死兆并提升终结倍率 | 开局随机标记 2 个部位。击碎首个标记 → 获得 100 死兆 + 暴伤 ×3。持有 ×3 buff 时击碎另一标记 → 暴伤 ×6。 |

## 四、连击系统重构

### 4.1 新计数器模型

```
总连击 x47   追击 78/100   破甲 22/100   2s
```

| 计数器 | 说明 | 显示条件 | 归零条件 |
|--------|------|---------|---------|
| 总连击 | 每次点击 +1，无限叠加 | 始终显示 | 5s 无点击归零 |
| 追击进度 | 均衡攻势专属计数 | 已学 `normal_core` | 到 100 触发追击爆发后归零 |
| 破甲进度 | 碎盾攻坚专属计数 | 已学 `armor_core` | 到 100 触发崩塌后归零 |

**统一规则**：
- 所有计数共享一个 5 秒超时计时器
- 任意点击重置所有计时器
- 特殊计数器到阈值归零重算（不再保留 30% 蓄力返还进度——该机制改为内化到暴风连击本身的触发计数减少）
- 未学任何天赋时，只显示 `x{count}` 数字，无进度条

### 4.2 前端改动

| 文件 | 改动 |
|------|------|
| `publicPageState.js` | 拆分 `comboCount` → `totalCombo` + `stormCombo` + `armorCombo` |
| `publicPageState.js` | 新增 `comboCounters` computed，返回需要显示的行 |
| `BattlePage.vue` | 连击行改为多计数器展示 |
| `BattlePage.vue` | 暴风连击触发后 `stormCombo` 归零而非 ×0.3 |

### 4.3 连击行模板示意

```html
<div class="combo-counter">
  <span class="combo-counter__total">x{{ totalCombo }}</span>
  <span v-if="hasStorm" class="combo-counter__track combo-counter__track--storm">
    追击 {{ stormCombo }}/{{ stormTrigger }}
    <span class="combo-counter__track-bar" :style="{ width: stormPercent + '%' }"></span>
  </span>
  <span v-if="hasArmor" class="combo-counter__track combo-counter__track--armor">
    破甲 {{ armorCombo }}/{{ armorTrigger }}
    <span class="combo-counter__track-bar" :style="{ width: armorPercent + '%' }"></span>
  </span>
  <span class="combo-counter__reset">{{ resetSeconds }}s</span>
</div>
```

## 六、动画效果修复：定位到对应格子

### 6.1 当前问题

部分效果（`storm_combo` 绿色刀光、`auto_strike` 锤击、`bleed` 血滴、`omen_harvest` 镰刀、`final_cut` 巨刃）使用**固定 CSS 位置**（`top: 50%; left: 50%`），始终出现在容器中央。应该定位到**触发该效果的具体格子**上。

### 6.2 修复方案

已有基础设施：
- `findBossZoneElement(partX, partY)` → 找到对应格子的 DOM 元素
- `triggerAnchor(type)` → 返回 `{left, top}` 相对于 overlay 容器的偏移
- `triggerOverlayStyle(type, options)` → 返回完整的 `position: absolute; left/right/top/bottom` 样式对象
- `collapse_trigger` 已经在使用这套定位（line 513-527）

需要修改的效果：

| 效果 | 当前 | 改为 |
|------|------|------|
| `storm_combo` 刀光 | `top: 50%; left: 50%` | `triggerOverlayStyle('storm_combo')` |
| `auto_strike` 锤击 | 固定 className `talent-effect-overlay__hammer` | 加 `:style="triggerOverlayStyle('auto_strike')"` |
| `bleed` 血滴 | 固定 className | 加 `:style="triggerOverlayStyle('bleed')"` |
| `omen_harvest` 镰刀 | `top: 50%; left: 50%` | `triggerOverlayStyle('omen_harvest')` |
| `final_cut` 巨刃 | `top: 50%; left: 50%` | `triggerOverlayStyle('final_cut')` |
| `judgment_day` | 固定位置 | `triggerOverlayStyle('judgment_day')` |
| `silver_storm` | 固定位置 | 全屏效果保留居中 |

### 6.3 额外检查

确保后端 `TalentTriggerEvent` 在发送这些事件时携带 `PartX` / `PartY` 字段。若当前未携带，补上。当前 `collapse_trigger` 已带，需检查 `storm_combo`、`auto_strike`、`bleed`、`omen_harvest`、`final_cut`。
