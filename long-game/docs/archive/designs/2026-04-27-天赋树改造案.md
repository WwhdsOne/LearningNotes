# 天赋树改造案 — 层锁 + 小节点 + 层满奖励（2026-04-27）

## 背景

基于现有三系五层天赋结构（均衡攻势 / 碎盾攻坚 / 致命洞察），新增以下机制：

1. **层锁规则**：学习下一层前必须点满当前层所有节点（含小节点）
2. **小加成节点**：在现有主节点之间插入小属性节点
3. **层满奖励**：每点满一层给予一个全局被动奖励

数值约束：不膨胀，每系小节点总增幅不超过 30%，层满奖励不超过 15%。

---

## 一、层锁规则

### 规则定义

- 主系必须点满第 N 层的所有节点（主节点 + 小节点），才能学习第 N+1 层的任意节点
- 副系不受此限制（副系最多学 2 个中层节点，不可能点满整层）
- Tier 4（终极层）点满后无下一层，无额外锁

### 受影响的函数

| 位置 | 函数 | 改动 |
|------|------|------|
| `talent.go` | `Store.LearnTalent` | 新增前置层满校验 |
| `TalentsPage.vue` | `canLearn()` / `nodeState()` | 新增 `layer-locked` 状态 |
| `TalentsPage.vue` | `stateReason()` | 新增"需先点满第 N 层"提示 |

### 点满判定

某层点满 = 该层所有 `id` 在 `talentState.talents` 中。

```go
func isTierFull(tier int, learned []string) bool {
    needed := tierNodeCount[tier] // 主节点 + 小节点总数
    learnedCount := 0
    for _, id := range learned {
        if talentDefs[id].Tier == tier {
            learnedCount++
        }
    }
    return learnedCount >= needed
}
```

## 二、小加成节点

### 分布方案

每系插入 6 个小节点，分布在 Tier 1~3：

| 层级 | 主节点数 | 小节点数 | 小节点属性 | 小节点成本 |
|------|---------|---------|-----------|-----------|
| Tier 0 | 1 (核心) | 0 | — | — |
| Tier 1 | 3 | **2** | +3% 攻击力 / +3% 对应系伤害 | 120 |
| Tier 2 | 3 | **2** | +3% 暴伤或穿甲 / +3% 对应系伤害 | 150 |
| Tier 3 | 2 | **2** | +5% 首领伤害 / +3% 对应系伤害 | 200 |
| Tier 4 | 1 (终极) | 0 | — | — |

小节点不计入副系学习限制计数（副系仍为"最多 2 个中层节点"）。

### 前端坐标

小节点沿当前半圆弧线分布在主节点之间。以 Tier 1（原本 3 个节点均匀分布在 135°→45° 弧上）为例，插入 2 个小节点后变为：

```
主节点1 — 小节点 — 主节点2 — 小节点 — 主节点3
```

坐标算法：`coordinatesForTierNodes` 改为按 `(主 + 小)` 总数均匀分布，小节点渲染为稍小的圆形（`width: 72px` vs 主节点 `92px`）。

## 三、层满奖励

### 奖励定义

每系每层点满后自动获得一个永久被动奖励，不占天赋点、不占用节点槽位。

#### 均衡攻势（normal）

| 层 | 奖励效果 | 后端 effectType | 数值 |
|----|---------|----------------|------|
| T0 | +5% 全伤害 | `all_damage_amplify` | 0.05 |
| T1 | +8% 攻击力 | `attack_power_percent` | 0.08 |
| T2 | 暴风连击触发要求 -20 次 | `storm_trigger_reduce` | 20 |
| T3 | +10% 首领伤害 | `boss_damage` | 0.10 |
| T4 | 白银风暴额外 +3 段 | `storm_extra_hits` | 3 |

#### 碎盾攻坚（armor）

| 层 | 奖励效果 | 后端 effectType | 数值 |
|----|---------|----------------|------|
| T0 | +5% 全伤害 | `all_damage_amplify` | 0.05 |
| T1 | 崩摧触发需求 -30 次 | `collapse_trigger_reduce` | 30 |
| T2 | +10% 护甲穿透 | `armor_pen_extra` | 0.10 |
| T3 | 崩摧状态 +10% 易伤 | `collapse_vulnerability` | 0.10 |
| T4 | 审判日削除比例 +5% | `judgment_day_boost` | 0.05 |

#### 致命洞察（crit）

| 层 | 奖励效果 | 后端 effectType | 数值 |
|----|---------|----------------|------|
| T0 | +5% 全伤害 | `all_damage_amplify` | 0.05 |
| T1 | +5% 暴击率 | `crit_rate_bonus` | 0.05 |
| T2 | 斩杀血线 +3% | `omen_kill_threshold_raise` | 0.03 |
| T3 | +0.2% 每层死兆暴伤 | `omen_crit_damage` | 0.002 |
| T4 | 末日审判暴伤倍率 +1x | `doom_mult_boost` | 1.0 |

### 存储方案

层满奖励作为 `TalentModifiers` 的一部分返回，不单独存储。在 `ComputeTalentModifiers` 中根据已学节点按层检测，自动累加奖励加成。

```go

    // 每层检测：该层所有节点是否全部在 learned 中
    // 是 → 累加对应奖励到 modifiers
}
```

## 四、经济影响

### 点满一系的总成本

| 阶段 | 原方案（仅主节点） | 新方案（含小节点 + 层满） |
|------|------------------|------------------------|
| Tier 0 | 100 | 100 |
| Tier 1 | 2000 × 3 = 6000 | 2000 × 3 + 120 × 2 = **6240** |
| Tier 2 | 8000 × 3 = 24000 | 8000 × 3 + 150 × 2 = **24300** |
| Tier 3 | 30000 × 2 = 60000 | 30000 × 2 + 200 × 2 = **60400** |
| Tier 4 | 120000 | 120000 |
| **总计** | **210,100** | **211,040** |

小节点总成本仅增加约 **940** 天赋点（不足 1%），对整体经济影响极小。

### 层锁带来的强制成本

由于必须点满当前层才能进下一层，无法再"跳过"高层节点直奔终极。实际上原方案中：
- Tier 0 → Tier 4 直冲成本：100 + 2000 + 8000 + 30000 + 120000 = **160,100**
- 新方案强制成本：需点满所有层 → **211,040**

终极层的实际获取成本提高了约 **32%**。如果后续发现节奏偏慢，可适当上调天赋点投放（如 Boss 击杀天赋点 +10%~20%）。
