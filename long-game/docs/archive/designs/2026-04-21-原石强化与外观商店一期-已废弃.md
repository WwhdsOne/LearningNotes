# 2026-04-21 原石强化与外观商店一期

## 上线范围

本期同时上线两条长期养成线：

- 原石强化：重复装备、重复英雄先分解成原石，再投入装备强化、英雄觉醒和外观购买。
- 外观商店：只上 4 套视觉外观，全部拆件售卖，只改轨迹与点击特效，不改任何数值。

## 原石闭环

- 分解装备：每消耗 `1` 件装备获得 `1` 原石。
- 分解英雄：每消耗 `1` 个重复英雄获得 `1` 原石。
- 如果装备当前穿戴中，则该类型至少保留 `1` 件。
- 如果英雄当前出战中，则该英雄至少保留 `1` 个。
- 原石当前统一用于：
  - 装备强化
  - 英雄觉醒
  - 外观购买

## 装备强化

- 接口：`POST /api/equipment/{itemId}/reforge`
- 消耗：每次 `20` 原石
- 目标：仅允许对已拥有的装备类型强化
- 写入：继续复用 `vote:user-equip-upgrade:<nickname>:<itemId>`

概率表：

- `52%`：BonusClicks +1
- `22%`：BonusCriticalCount +1
- `12%`：BonusCriticalChancePercent +1
- `13%`：随机两项 +1
- `1%`：大奖（三项都 +1）

保底：

- 连续 `30` 次未出大奖
- 第 `31` 次强制大奖
- 保底按“玩家 + 装备类型”独立记录
- 显示距离保底的进度条

## 英雄觉醒

- 接口：`POST /api/heroes/{heroId}/awaken`
- 消耗：每次 `25` 原石
- 目标：仅允许对已拥有英雄觉醒
- 写入：`vote:user-hero-upgrade:<nickname>:<heroId>`

觉醒字段：

- `awaken_level`
- `bonus_clicks`
- `bonus_critical_chance_percent`
- `bonus_critical_count`
- `trait_value`
- `pity_counter`

概率表：

- `36%`：BonusClicks +1
- `22%`：BonusCriticalCount +1
- `15%`：BonusCriticalChancePercent +1
- `14%`：TraitValue +1
- `12%`：随机两项 +1（四选二）
- `1%`：大奖（四项都 +1）

保底：

- 连续 `30` 次未出大奖
- 第 `31` 次强制大奖
- 保底按“玩家 + 英雄类型”独立记录

英雄觉醒后的最终属性同时作用于：

- 平时点击
- Boss 伤害

## 外观商店

一期上架 8 个单件商品：

- `trail-ribbon`：流星彩带轨迹
- `impact-ribbon`：流星彩带点击特效
- `trail-confetti`：纸片庆典轨迹
- `impact-confetti`：纸片庆典点击特效
- `trail-stamp`：印章敲击轨迹
- `impact-stamp`：印章敲击点击特效
- `trail-firefly`：流萤追光轨迹
- `impact-firefly`：流萤追光点击特效

统一规则：

- 每个轨迹单件 `30` 原石
- 每个点击特效单件 `30` 原石
- 不卖套装
- 不做整套折扣
- 允许混搭：`1` 个轨迹 + `1` 个点击特效

接口：

- `GET /api/shop`
- `POST /api/shop/cosmetics/{cosmeticId}/purchase`
- `POST /api/shop/cosmetics/equip`

## 视觉约束

- 外观只在前台个人视角生效，不影响别人画面。
- 外观不得改变按钮排序、倍率、标签和可点击性。
- 星光按钮系统态优先级高于外观；星光态下外观会自动降透明度并缩短持续时间。
- 自动挂机沿用同套外观，但使用低粒子简化版。
- 点击特效中心区域必须避开按钮文案与计数。

四套当前表现：

- 流星彩带：多彩丝带拖尾 + 彩带炸开
- 纸片庆典：彩纸碎片聚拢 + 彩屑喷散
- 印章敲击：短墨迹拖尾 + 盖章压印
- 流萤追光：暖光粒子游走 + 环形萤火散开
