# 2026-04-23 特殊小小英雄被动设计案

## 1. Summary

- 本文只定义特殊小小英雄被动的设计方向与样例，不做本轮实装。
- 小小英雄仍然是玩家最终构筑的一部分，但不进入技能树本体。
- 英雄被动只负责和战斗套路联动，不负责承载玩家长期成长。
- 当前项目仍是“单 Boss + 手动点击”结构，因此所有被动都必须围绕点击、暴击、连击、残血收尾来设计。

## 2. 设计原则

### 2.1 和技能树分层

- 技能树归玩家本人。
- 被动归英雄模板。
- 技能树决定战斗规则的长期成长方向。
- 英雄决定在某个方向上补一段特色收益。

### 2.2 不碰英雄属性树

- 不做“英雄专属技能树”。
- 不做“给英雄属性加点”的被动设计。
- 不做“靠觉醒把被动一路叠爆”的前置假设。

### 2.3 被动要能独立挂接

每个英雄被动都应该可以表达为：

- `passiveId`
- `triggerHook`
- `effectType`
- `params`
- `synergyTags`

要求：

- 新英雄被动接入时，不修改战斗核心主流程。
- 未识别的被动类型安全忽略，不打断战斗。

## 3. 通用内部约定

### 3.1 可复用 Hook

建议复用和技能树一致的触发阶段：

- `onClickBefore`
- `onDamageCalculate`
- `onCritical`
- `onDamageFinal`
- `onBossLowHpCheck`

### 3.2 内部套路标签

仅供内部设计和处理器路由使用：

- `crit`
- `combo`
- `execute`

玩家文案不直接显示这些标签。

### 3.3 BattleContext 读取范围

被动后续可以读取：

- `currentHp`
- `maxHp`
- `damage`
- `isCritical`
- `comboCount`
- `clickCountInWindow`

## 4. 被动类型分组

### 4.1 暴击联动型

定位：

- 不直接提高英雄面板成长。
- 强化“暴击已经发生之后”的收益。

适合承担的效果：

- 暴击时追加倍率
- 暴击后短时强化
- 暴击后对终局结算追加修正

### 4.2 连击联动型

定位：

- 围绕玩家连续点击的节奏感。
- 让连击路线不只靠技能树本身，也能被英雄拉出差异。

适合承担的效果：

- 连击启动更快
- 连击衰减更慢
- 连击到阈值后追加收益

### 4.3 斩杀联动型

定位：

- 围绕 Boss 残血区间设计收尾型收益。
- 控制触发窗口，避免过早越级秒杀。

适合承担的效果：

- 扩大斩杀血线
- 残血增伤
- 达到条件时触发特殊收尾结算

## 5. 首批特殊小小英雄被动草案

以下均为设计样例，本轮不实装。

### 5.1 暴击向英雄

#### 英雄：雷鸣鼓手

- 英雄定位：暴击爆发辅助
- 被动名称：`震拍回响`
- 触发阶段：`onCritical`
- 效果描述：暴击触发后，当前这次伤害获得一次额外倍率修正。
- 内部 `effectType`：`crit_damage_echo`
- `params` 示例：`{ "multiplier": 0.35 }`
- `synergyTags`：`["crit"]`
- 联动方向：适合和暴击深化路线一起使用。
- 风险备注：倍率不能过高，避免普通暴击直接越级。

#### 英雄：焰纹隼

- 英雄定位：暴击后续追打
- 被动名称：`热痕追咬`
- 触发阶段：`onDamageFinal`
- 效果描述：如果本次为暴击，则下一小段时间内的点击获得额外伤害修正。
- 内部 `effectType`：`crit_followup_window`
- `params` 示例：`{ "durationMs": 1800, "bonusPercent": 12 }`
- `synergyTags`：`["crit"]`
- 联动方向：适合高频点击和暴击连续触发场景。
- 风险备注：需要限制刷新方式，避免无限覆盖。

### 5.2 连击向英雄

#### 英雄：节拍猫

- 英雄定位：连击启动器
- 被动名称：`轻拍预热`
- 触发阶段：`onClickBefore`
- 效果描述：短时间连续点击时，连击计数更容易成立。
- 内部 `effectType`：`combo_bootstrap_bonus`
- `params` 示例：`{ "windowMs": 1200, "bonusCombo": 1 }`
- `synergyTags`：`["combo"]`
- 联动方向：适合连击路线的起手阶段。
- 风险备注：不应直接送高层数，只负责降低启动门槛。

#### 英雄：疾步狐

- 英雄定位：连击维持器
- 被动名称：`尾拍续节`
- 触发阶段：`onDamageCalculate`
- 效果描述：连击即将中断时，保留部分连击值并延长维持时间。
- 内部 `effectType`：`combo_decay_guard`
- `params` 示例：`{ "preserveCount": 3, "extendMs": 900 }`
- `synergyTags`：`["combo"]`
- 联动方向：适合中长线连击保持。
- 风险备注：不能让连击永久不掉，只能缓冲。

#### 英雄：街机熊

- 英雄定位：连击阈值爆发
- 被动名称：`喝彩拍点`
- 触发阶段：`onDamageFinal`
- 效果描述：当连击达到指定层数后，后续点击获得阶段性增伤。
- 内部 `effectType`：`combo_threshold_burst`
- `params` 示例：`{ "comboRequired": 12, "bonusPercent": 18 }`
- `synergyTags`：`["combo"]`
- 联动方向：适合已成型的连击路线。
- 风险备注：阈值和收益要一起控，不能过早爆发。

### 5.3 斩杀向英雄

#### 英雄：断刃鸦

- 英雄定位：低血线收尾
- 被动名称：`残翼切线`
- 触发阶段：`onBossLowHpCheck`
- 效果描述：Boss 进入残血阶段时，扩大收尾判定窗口。
- 内部 `effectType`：`execute_threshold_shift`
- `params` 示例：`{ "extraThresholdPercent": 3 }`
- `synergyTags`：`["execute"]`
- 联动方向：适合斩杀路线中后段。
- 风险备注：只能小幅抬高阈值，避免直接跳过正常战斗区间。

#### 英雄：白影猎犬

- 英雄定位：斩杀增伤
- 被动名称：`闻血前扑`
- 触发阶段：`onDamageCalculate`
- 效果描述：Boss 低于指定血线时，额外提高本次伤害。
- 内部 `effectType`：`boss_low_hp_damage_bonus`
- `params` 示例：`{ "hpPercent": 12, "bonusPercent": 20 }`
- `synergyTags`：`["execute"]`
- 联动方向：适合稳定收尾，不依赖一次性秒杀。
- 风险备注：优先做百分比增伤，不直接做硬斩杀。

#### 英雄：终铃修女

- 英雄定位：特殊收尾判定
- 被动名称：`终祷钟声`
- 触发阶段：`onDamageFinal`
- 效果描述：当 Boss 血量低于极低阈值时，本次命中触发一次特殊收尾结算。
- 内部 `effectType`：`boss_execute_by_max_hp`
- `params` 示例：`{ "hpPercent": 5 }`
- `synergyTags`：`["execute"]`
- 联动方向：作为斩杀路线的终局英雄。
- 风险备注：必须严控血线和触发条件，只允许极低阈值出现。

## 6. 可补充的中性英雄方向

这类英雄不直接站队某一条路线，但可以服务多个方向：

- 首次暴击后提供短时节奏加速
- Boss 进入半血后给出稳定输出修正
- 连击和暴击都成立时触发小幅联动奖励

建议中性英雄只做少量，用于过渡，不应抢走专精英雄的定位。

## 7. 配表建议

后续英雄模板中建议把被动配置成独立结构，而不是继续扩 `traitType`：

| 字段 | 含义 |
| --- | --- |
| `passiveId` | 被动唯一标识 |
| `displayName` | 前端展示名 |
| `description` | 前端展示描述 |
| `triggerHook` | 触发阶段 |
| `effectType` | 内部效果类型 |
| `params` | 参数配置 |
| `synergyTags` | 内部联动标签 |

兼容建议：

- 旧 `traitType` / `traitValue` 继续只做兼容读取。
- 新特殊英雄被动不再依赖旧字段继续扩展。

## 8. 当前阶段限制

- 本轮不把这些被动接入战斗。
- 本轮不开放给玩家选择。
- 本轮不做英雄被动数值定版。
- 本轮不做英雄与技能树的 UI 联动。
- 本轮不做多英雄组合叠加平衡。

## 9. 一句话结论

特殊小小英雄的价值不在于进入技能树，而在于站在技能树之外，为暴击、连击、斩杀这些内部套路方向补上一段有辨识度的联动收益；本轮先把被动类型、触发时机和样例英雄写进 docs，后续再接到底层效果系统。
