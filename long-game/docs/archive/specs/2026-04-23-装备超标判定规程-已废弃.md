# 2026-04-23 装备超标判定规程

## 1. 用途

本文给审表工具或弱模型使用，只解决一个问题：判断一件装备实例设计是否符合新装备体系。

固定输出三档：

- `合格`
- `临界`
- `超标`

除这三档外，不输出第四种结论。

本文必须和 [装备稀有度与成长数值规范](2026-04-23-装备稀有度与成长数值规范.md) 保持一致。

## 2. 输入字段

每次只判断一件装备：

```json
{
  "rarity": "传说",
  "enhanceLevel": 20,
  "breakthroughLevel": 0,
  "quality": "perfect",
  "affixes": [
    {"category": "specialization", "tree": "armor_break", "name": "贯穿", "tier": "T1"},
    {"category": "general", "tree": null, "name": "攻击力", "tier": "T2"},
    {"category": "exclusive", "tree": "armor_break", "name": "审判加速", "tier": "legendary"}
  ],
  "baseStats": {
    "attack": 100
  }
}
```

字段要求：

| 字段 | 要求 |
| --- | --- |
| `rarity` | 只能是普通、稀有、史诗、传说、神话、至臻 |
| `enhanceLevel` | 大于等于 0 的整数 |
| `breakthroughLevel` | 大于等于 0 的整数 |
| `quality` | 只能是 `normal`、`fine`、`perfect`、`ancient` |
| `affixes` | 数组，数量必须符合稀有度槽位 |
| `baseStats` | 至少存在一个非负数值属性 |

字段缺失、枚举非法、数字为负，直接输出 `超标`，原因写 `输入字段非法`。

## 3. 基准表

### 3.1 稀有度表

| 稀有度 | 词缀总数 | 基础强化上限 | 最大突破等级 | 最大强化上限 | 是否需要专属词缀 |
| --- | --- | --- | --- | --- | --- |
| 普通 | 0 | 5 | 0 | 5 | 否 |
| 稀有 | 1 | 10 | 0 | 10 | 否 |
| 史诗 | 2 | 15 | 0 | 15 | 否 |
| 传说 | 3 | 20 | 1 | 40 | 是 |
| 神话 | 4 | 25 | 1 | 60 | 是 |
| 至臻 | 5 | 30 | 2 | 100 | 是 |

### 3.2 突破后强化上限

| 稀有度 | `breakthroughLevel` | 允许强化上限 |
| --- | --- | --- |
| 普通 | 0 | 5 |
| 稀有 | 0 | 10 |
| 史诗 | 0 | 15 |
| 传说 | 0 | 20 |
| 传说 | 1 | 40 |
| 神话 | 0 | 25 |
| 神话 | 1 | 60 |
| 至臻 | 0 | 30 |
| 至臻 | 1 | 80 |
| 至臻 | 2 | 100 |

### 3.3 品质表

| 稀有度 | 允许品质 |
| --- | --- |
| 普通 | `normal`、`fine` |
| 稀有 | `normal`、`fine`、`perfect` |
| 史诗 | `fine`、`perfect`、`ancient` |
| 传说 | `perfect`、`ancient` |
| 神话 | `perfect`、`ancient` |
| 至臻 | `ancient` |

### 3.4 词缀分类

| 分类 | `category` | 规则 |
| --- | --- | --- |
| 通用词缀 | `general` | 可与任意专精共存 |
| 专精词缀 | `specialization` | 同一件装备只能出现一个流派 |
| 专属词缀 | `exclusive` | 只允许传说、神话、至臻出现，且必须与专精流派一致 |

允许的流派：

- `basic_attack`
- `armor_break`
- `critical`

禁止词缀名称关键词：

- 攻击速度
- 点击间隔
- 额外点击
- 自动点击概率
- 每秒点击

## 4. 固定判定步骤

严格按顺序判断。

### 第一步：字段合法性

若字段缺失、枚举非法、数值为负、数组类型错误，直接判 `超标`。

### 第二步：稀有度槽位检查

若 `affixes.length` 不等于稀有度要求的词缀总数，判 `超标`。

普通装备必须为 0 条词缀。

### 第三步：强化上限检查

按稀有度和 `breakthroughLevel` 查允许强化上限。

以下情况判 `超标`：

- `breakthroughLevel` 超过该稀有度最大突破等级。
- `enhanceLevel` 超过当前突破等级允许强化上限。
- 普通、稀有、史诗出现 `breakthroughLevel > 0`。

以下情况记为 `临界候选`：

- `enhanceLevel` 已达到当前允许强化上限。
- 传说、神话、至臻已突破且强化等级超过基础强化上限。

### 第四步：品质检查

若 `quality` 不在该稀有度允许品质中，判 `超标`。

若非至臻装备使用 `ancient` 品质，记为 `临界候选`。

### 第五步：词缀分类检查

以下情况判 `超标`：

- 词缀 `category` 不在 `general`、`specialization`、`exclusive` 中。
- 词缀 `tree` 使用未知流派。
- 传说、神话、至臻没有且仅有 1 条 `exclusive` 词缀。
- 普通、稀有、史诗出现 `exclusive` 词缀。
- 出现多个不同流派的 `specialization` 词缀。
- `exclusive` 词缀流派与专精词缀流派不一致。

以下情况记为 `临界候选`：

- 史诗装备的 2 条词缀全部为同一流派专精。
- 神话或至臻装备除专属词缀外，全部常规槽位都为同一流派专精。

### 第六步：禁止词缀检查

若任一词缀名称包含禁止关键词，判 `超标`。

### 第七步：基础属性检查

`baseStats` 中所有数值必须为非负数。

以下情况判 `超标`：

- 无任何基础属性。
- 任一基础属性为负数。
- 普通装备基础属性倍率或品质效果被写成高于精良上限。

以下情况记为 `临界候选`：

- 至臻以外装备同时拥有远古品质和满强化。
- 传说或神话装备同时达到突破后强化上限，并拥有全部 T1 常规词缀。

### 第八步：输出结论

- 只要任一步命中 `超标`，最终输出 `超标`。
- 否则，只要存在任意 `临界候选`，最终输出 `临界`。
- 否则输出 `合格`。

## 5. 输出格式

固定输出 JSON：

```json
{
  "verdict": "临界",
  "reasons": [
    "装备已达到当前强化上限",
    "非至臻装备使用远古品质"
  ],
  "violations": [],
  "advice": "若要降低风险，建议下调品质或保留突破后强化空间"
}
```

字段要求：

- `verdict` 只能是 `合格`、`临界`、`超标`。
- `reasons` 写触发结论的直接原因。
- `violations` 只写硬超标项；没有就给空数组。
- `advice` 只给 1 句修正建议。

## 6. 示例

### 6.1 合格：史诗普攻装

```json
{
  "rarity": "史诗",
  "enhanceLevel": 12,
  "breakthroughLevel": 0,
  "quality": "fine",
  "affixes": [
    {"category": "specialization", "tree": "basic_attack", "name": "牢固", "tier": "T1"},
    {"category": "general", "tree": null, "name": "攻击力", "tier": "T2"}
  ],
  "baseStats": {"attack": 100}
}
```

结论：`合格`。

### 6.2 临界：满强化传说装

```json
{
  "rarity": "传说",
  "enhanceLevel": 40,
  "breakthroughLevel": 1,
  "quality": "ancient",
  "affixes": [
    {"category": "specialization", "tree": "armor_break", "name": "贯穿", "tier": "T1"},
    {"category": "general", "tree": null, "name": "全伤害增幅", "tier": "T1"},
    {"category": "exclusive", "tree": "armor_break", "name": "审判加速", "tier": "legendary"}
  ],
  "baseStats": {"attack": 160}
}
```

结论：`临界`，原因是已达到突破后强化上限且使用远古品质。

### 6.3 超标：稀有装备出现专属词缀

```json
{
  "rarity": "稀有",
  "enhanceLevel": 5,
  "breakthroughLevel": 0,
  "quality": "fine",
  "affixes": [
    {"category": "exclusive", "tree": "critical", "name": "末日预兆", "tier": "legendary"}
  ],
  "baseStats": {"attack": 80}
}
```

结论：`超标`，原因是稀有装备不允许专属词缀。

### 6.4 超标：跨流派专精

```json
{
  "rarity": "神话",
  "enhanceLevel": 25,
  "breakthroughLevel": 0,
  "quality": "perfect",
  "affixes": [
    {"category": "specialization", "tree": "basic_attack", "name": "牢固", "tier": "T1"},
    {"category": "specialization", "tree": "critical", "name": "残暴", "tier": "T1"},
    {"category": "general", "tree": null, "name": "攻击力", "tier": "T2"},
    {"category": "exclusive", "tree": "critical", "name": "死兆永存", "tier": "mythic"}
  ],
  "baseStats": {"attack": 180}
}
```

结论：`超标`，原因是同一装备同时出现普攻和暴击专精。

## 7. 一句话判定模板

先校验字段，再查稀有度槽位、突破上限、品质范围、词缀分类、流派互斥和禁止词缀；任何硬规则违规输出 `超标`，否则存在高风险组合输出 `临界`，其余输出 `合格`。
