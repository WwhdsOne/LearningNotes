# 伤害计算链路总览（暴击 / 天赋 / 装备）

本文用于开发排查“点击到伤害入账”的核心调用链，重点覆盖暴击、天赋、装备三条线。

## 1. 请求入口与主调用链

1. 点击接口入口：`registerButtonClickRoutes` 处理 `POST /api/buttons/:slug/click`
   - 文件：`backend/internal/httpapi/button_click_routes.go:12,58`
2. 点击执行分发：`executeButtonClick`
   - 文件：`backend/internal/httpapi/realtime_helpers.go:178`
   - 说明：启用手动点击票据时会走 `ManualClickService.Click`，否则直接调用 `Store.ClickButton`。
3. 核心结算入口：`Store.ClickButton`
   - 文件：`backend/internal/vote/store.go:650`
   - 说明：
     - 无 Boss：走 `applyVoteOnlyClick`（点击数/榜单更新）
     - 有 Boss：统一走 `applyBossPartClick` / `applyBossPartDamage`（Go 公式结算）
     - 无部位 Boss 已被限制为非法状态，创建阶段会直接拒绝

## 2. 暴击与普通点击伤害链路

1. 点击增量计算：`nextIncrement`
   - 文件：`backend/internal/vote/store.go:1442`
2. `nextIncrement` 内部流程：
   - `inventoryQuantities` 读取背包数量（`store.go`）
   - `loadoutForNickname` 组装已穿戴装备（`store.go:1716`）
   - `combatStatsForNickname` 生成当前战斗属性（`store.go:1474`）
3. 战斗属性构建：
   - `baseCombatStats`（基础值）`store.go:1486`
   - `loadoutBonuses`（叠装备攻击/破甲/暴击率/暴伤）`store.go:1497`
   - `deriveCombatStats`（推导普通伤害/暴击伤害）`store.go:1517`
4. 暴击随机：`criticalRollPlan`
   - 文件：`backend/internal/vote/store.go:2446`
   - 说明：根据暴击率决定随机区间与阈值。
5. 装备暴击率折算：
   - `critRate` 存储为 0~1 小数（如 0.05）
   - 在 `combatStatsForNickname` 中折算为百分比并累加到 `CriticalChancePercent`：`critRate * 100`

## 3. Boss 伤害链路

### 3.1 无部位 Boss（已禁用）

当前版本已禁止“无部位 Boss”创建与激活。若出现无部位数据会返回 `ErrBossPartsRequired`，不再进入点击结算。

### 3.2 多部位 Boss

1. 手动点部位入口：`ClickBossPart` / `clickBossPart`
   - 文件：`backend/internal/vote/store.go:731,1134`
2. 自动挂机入口：`AutoClickBossPart`
   - 文件：`backend/internal/vote/store.go:1114`
3. 部位伤害结算：`applyBossPartDamage`
   - 文件：`backend/internal/vote/store.go:1174`
4. 公式核心：`CalcBossPartDamage`
   - 文件：`backend/internal/vote/store.go:1539`
   - 当前公式要点：
     - 攻击力：`atk = max(1, AttackPower)`
     - 部位系数：`PartType.DamageCoefficient()`（soft=1.0/heavy=0.4/weak=2.5）
     - 有效护甲：`partArmor * (1 - ArmorPenPercent)`（破甲上限 80%）
     - 基础伤害：`max(atk*系数 - 护甲, 1)`
     - 增伤：`AllDamageAmplify`
     - 暴击：再次按暴击率随机，命中后乘 `CritDamageMultiplier`

## 3.3 Boss 创建限制（新增）

1. 手动激活 Boss 必须带部位
   - `ActivateBoss`：`backend/internal/vote/admin.go`
2. Boss 池模板必须带布局部位
   - `SaveBossTemplate`：`backend/internal/vote/boss_pool.go`
3. 从模板激活与写入当前 Boss 时再次校验
   - `activateBossTemplateInstance`、`setCurrentBoss`：`backend/internal/vote/boss_pool.go`
4. 对外错误
   - 错误常量：`ErrBossPartsRequired`（`backend/internal/vote/store.go`）
   - 管理后台接口返回：`400 BOSS_PARTS_REQUIRED`（`backend/internal/httpapi/admin_boss_routes.go`）

## 4. 装备链路（来源 -> 穿戴 -> 生效）

1. 装备模板维护（后台）
   - `SaveEquipmentDefinition` / `ListEquipmentDefinitions`
   - 文件：`backend/internal/vote/admin.go:93,64`
2. 穿戴与卸下
   - `EquipItem` / `UnequipItem`
   - 文件：`backend/internal/vote/store.go:740,782`
   - 接口：`/api/equipment/:itemId/equip`、`/api/equipment/:itemId/unequip`
   - 文件：`backend/internal/httpapi/equipment_routes.go:17,56`
3. 生效数据读取
   - `getEquipmentDefinition`（从 Redis 读模板）`store.go:1672`
   - `buildInventoryItem`（映射到背包/装备项）`store.go:2365`
   - `loadoutBonuses`（汇总到战斗属性）`store.go:1497`
4. 当前实际参与伤害计算的装备字段
   - `AttackPower`
   - `ArmorPenPercent`
   - `CritRate`
   - `CritDamageMultiplier`

### 重要说明（当前实现）

当前 `Store` 的伤害主链路（`nextIncrement`、`combatStatsForNickname`、`applyBossPartDamage`）中，未调用 `ComputeTalentModifiers` 与 `ApplyTalentEffectsToCombatStats`。

也就是说：
- 天赋系统的数据读写与定义已存在；
- 但天赋修正尚未真正接入当前伤害结算主链路。

## 6. 开发排查建议（按优先顺序）

1. 先看入口是否走到 `Store.ClickButton`：`realtime_helpers.go:178` -> `store.go:650`
2. 再看伤害来源分支：
   - 普通/无 Boss：`nextIncrement`（`store.go:1442`）
   - Boss（仅允许有部位）：`applyBossPartDamage` + `CalcBossPartDamage`
3. 若需排查“天赋为何不生效”，优先核对 `combatStatsForNickname`（`store.go:1474`）是否引入 `ComputeTalentModifiers`。
