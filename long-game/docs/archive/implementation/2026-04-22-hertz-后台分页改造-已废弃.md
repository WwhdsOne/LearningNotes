# 2026-04-22 Hertz 与后台分页改造

## 改造目标

- 后端运行时从标准库 HTTP 服务切到 Hertz 承载。
- 管理后台避免一次性拉取全量按钮、装备和历史 Boss，降低首屏和切页压力。
- 保持前台 API、Boss 规则、Redis 数据模型和 Consul 配置来源不变。

## 后端变化

- 服务入口改为 Hertz 启动，`/debug/pprof/` 改由 Hertz 中间件挂载。
- 现有 `httpapi` 路由继续复用，先通过 Hertz adaptor 承载，保证现有公开接口和 SSE 行为不变。
- `GET /api/admin/state` 收缩为后台摘要，只保留当前 Boss、排行榜、掉落快照、Boss 池、玩家统计等轻量数据。
- 新增后台分页读取：
  - `GET /api/admin/buttons?page=1&pageSize=20`
  - `GET /api/admin/equipment?page=1&pageSize=20`
  - `GET /api/admin/boss/history?page=1&pageSize=20`

## 数据层变化

- `vote.Store` 新增后台分页结果类型：
  - `AdminButtonPage`
  - `AdminEquipmentPage`
  - `AdminBossHistoryPage`
- 后台分页统一返回：
  - `items`
  - `page`
  - `pageSize`
  - `total`
  - `totalPages`
- 目前按钮、装备、历史 Boss 都采用传统页码分页，默认每页 20 条，最大 100 条。

## 前端变化

- 管理后台的按钮、装备、历史 Boss 视图改为传统分页。
- 登录后除了后台摘要，还会单独加载按钮页和装备页。
- 保存/删除按钮、保存/删除装备后，只刷新当前分页，不再刷新整个后台摘要。

## 验证

- `go -C backend test ./...`
- `npm --prefix frontend run build`
- `make check`
