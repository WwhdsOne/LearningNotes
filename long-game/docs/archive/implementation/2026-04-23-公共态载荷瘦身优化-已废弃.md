# 公共态载荷瘦身优化

## 目标

- 压缩首屏 `/api/buttons` 与 WS/SSE `public_state` / `public_delta` 的高频载荷。
- 不改玩法规则，只拆分高频公共态和低频公共资源。
- 首页按钮改为分页读取，第一页固定展示 9 个按钮。

## 本次落地

### 1. 公共态协议瘦身

- `vote.Snapshot` 只保留高频字段：
  - `buttons`
  - `buttonPage`
  - `buttonPageSize`
  - `buttonTotal`
  - `buttonTotalPages`
  - `totalVotes`
  - `leaderboard`
  - `boss`
  - `bossLeaderboard`
  - `starlight`
  - `announcementVersion`
- 公共态不再直接携带：
  - `bossLoot`
  - `bossHeroLoot`
  - `latestAnnouncement`

### 2. 按钮分页

- 新增 `GET /api/buttons/pages?page=1&pageSize=9`
- 第一页规则：
  - 先放当前星光位按钮
  - 再补普通按钮
  - 总数最多 9 个
- 第二页及之后：
  - 只返回普通按钮
  - 不再重复星光按钮
- 首页 `/api/buttons` 与实时公共态现在都只带第一页按钮，并额外返回分页元信息和总票数。

### 3. Boss 低频资源

- 新增 `GET /api/boss/resources`
- 返回当前 Boss 的：
  - `bossId`
  - `templateId`
  - `status`
  - `bossLoot`
  - `bossHeroLoot`
- 后端在 `events.Cache` 中加入 Boss 低频资源缓存。
- `boss_changed` 时同步刷新该缓存。

### 4. 公告缓存

- 公共态只广播 `announcementVersion`。
- 前端新增本地缓存：
  - 优先用缓存恢复 `latestAnnouncement`
  - 页面启动或版本变化时再请求 `/api/announcements/latest`

### 5. 前端行为调整

- `PublicPage.vue` 现在维护：
  - 当前页按钮
  - 首页按钮缓存
  - 按钮页码与总页数
  - 公告版本与公告正文缓存
  - Boss 资源按需拉取状态
- 搜索与标签只作用于当前页。
- 顶部按钮统计继续使用后端返回的总按钮数和总票数，不跟着当前页缩小。
