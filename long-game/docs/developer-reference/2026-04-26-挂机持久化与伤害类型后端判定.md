# 挂机持久化与伤害类型后端判定

## 1. 伤害类型改为后端判定

- `click_ack` 新增字段：
  - `payload.button.key`
  - `payload.bossDamage`
  - `payload.damageType`
- 后端判定入口：`backend/internal/vote/store.go`
  - `resolveBossDamageType(...)`
- WebSocket 回包组装：`backend/internal/httpapi/realtime_socket.go`

### 当前伤害类型值

- `doomsday`
- `judgement`
- `weakCritical`
- `critical`
- `trueDamage`
- `pursuit`
- `normal`

前端优先使用后端下发 `damageType`，不再按前端阈值做主判定。

## 2. 挂机状态改为 Redis 持久化

- 实现文件：`backend/internal/httpapi/afk_service.go`
- 服务初始化：`backend/cmd/server/main.go`

### 核心变化

1. 挂机玩家状态、累计战利品结算改为 Redis 存储，不再使用进程内 `map`。
2. 关闭页面后只要不再上报可见心跳，60 秒后会进入挂机。
3. 服务重启后可继续读取 Redis 状态并继续挂机循环。
4. 返回页面后会停止挂机并累积结算，前端调用 `/api/player/afk/settlement` 自动弹窗。

### Redis 键

- 玩家索引：`<redisPrefix>afk:players`
- 玩家挂机状态：`<redisPrefix>afk:player:<nickname>`
- 挂机结算暂存：`<redisPrefix>afk:settlement:<nickname>`

## 3. 前端规则展示与上报（不做本地判定）

- 展示位置：`frontend/src/pages/BattlePage.vue`
- 可见页心跳：`frontend/src/pages/publicPageState.js`
  - 心跳间隔常量：`AFK_HEARTBEAT_INTERVAL_MS`
  - 仅在页面可见时上报 `visible=true`
  - 页面隐藏时上报一次 `visible=false`

这保证前端只负责提示与上报，挂机开始/停止与结算由后端统一裁定。
