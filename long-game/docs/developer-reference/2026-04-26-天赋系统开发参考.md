## 1. 当前接口与调用链

### 1.1 后端

1. Boss 击杀发放资源
   - 入口：`Store.finalizeBossKill`
   - 行为：按 Boss 配置发放金币、强化石、天赋点（`talent_points`）
2. 升级天赋
   - API：`POST /api/talents/upgrade`
   - 存储：`Store.UpgradeTalent`
   - 行为：层锁校验 -> 点数检查 -> 写入 `{talentId: level}` -> 扣减点数
3. 洗点返还
   - API：`POST /api/talents/reset`
   - 存储：`Store.ResetTalents`
   - 行为：按每个已学节点的累计实际成本返还天赋点
4. 天赋定义与状态
   - `GET /api/talents/defs`：返回静态定义、基准成本、层满奖励文案
   - `GET /api/talents/state`：返回当前等级、天赋点、后端效果预览 `effectLines`
5. 战斗生效
   - 静态加成：`Store.ComputeTalentModifiers`
   - 触发型效果：`Store.applyTriggeredTalentDamage` 等战斗逻辑

### 1.2 前端

1. `TalentsPage.vue` 调用 `/api/talents/defs` 与 `/api/talents/state`
2. 点击升级走 `/api/talents/upgrade`
3. 升级成功后直接使用响应里的：
   - `talents`
   - `talentPoints`
   - `effectLines`
4. 前端不再自行推导天赋数值描述，只负责渲染后端预览

## 2. 当前存储模型

### 2.1 TalentState

当前已不是旧的 `[]string` 已学列表，而是等级映射：

```go
map[string]int // talentID -> level
```

说明：

- `0` 表示未学习
- `1~5` 表示当前等级
- 非基石节点只要求上一层所有节点达到 `Lv1`

### 2.2 成本模型

统一基准成本：

- Tier0 基石：`20`
- Tier1 主节点：`30`
- Tier2 主节点：`80`
- Tier3 主节点：`150`
- Tier4 终极：`200`
- Tier1 小节点：`15`
- Tier2 小节点：`35`
- Tier3 小节点：`60`

升级公式：

```text
cost(level) = round(baseCost × level^0.85 × 1.8)
```

关键口径：

- `cost(level)` 是“升到该等级这一次”的单次消耗
- `TalentLevelCostDiff(base, current, target)` 会把中间每一级逐级累加
- `TalentCumulativeCost(base, target)` 用于洗点返还累计值

## 3. 当前调参入口

### 3.1 成本与升级曲线

- 文件：`backend/internal/vote/talent.go`
- 入口：
  - `TalentCostTier*`
  - `TalentLevelCost`
  - `TalentLevelCostDiff`
  - `TalentCumulativeCost`

### 3.2 节点定义

- 文件：`backend/internal/vote/talent.go`
- 入口：`talentDefs`
- 说明：节点基准效果、层级、基准成本都从这里发源

### 3.3 展示预览

- 文件：`backend/internal/vote/talent.go`
- 入口：`BuildTalentEffectLines`
- 说明：天赋页浮层说明统一由后端生成，前端不再本地计算

### 3.4 实际战斗结算

- 文件：`backend/internal/vote/store.go`
- 说明：所有触发次数、持续时间、倍率、阈值的实际生效都应与 `BuildTalentEffectLines` 对齐

## 4. 当前联调重点

1. 升级扣点是否与前端显示一致
2. 洗点返还是否等于历史逐级支出
3. `/api/talents/state` 与 `/api/talents/upgrade` 是否都返回最新 `effectLines`
4. 关键成长型节点的“展示预览”和“实际战斗”是否一致
5. `silver_storm` 是否按时间窗而不是攻击次数递减

## 5. 已废弃口径

以下内容已废弃，不要继续沿用：

- `POST /api/talents/learn`
- `Store.LearnTalent`
- 单节点前置规则（如“必须先学某个指定天赋”）
- 固定学费 `talent.cost`
- `talentPoints -= talent.cost` 这一条旧扣点公式
- `TalentState` 的已学字符串数组模型
- 前端本地计算天赋效果文案
- 固定成本 `100 / 2000 / 8000 / 30000 / 120000`
