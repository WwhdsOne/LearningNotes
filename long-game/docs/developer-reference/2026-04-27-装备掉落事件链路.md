# 装备掉落事件链路

> 从攻击 Boss 到装备入袋的完整后端链路，涵盖手动点击和挂机两种入口。

## 涉及文件

| 文件 | 职责 |
|---|---|
| `internal/httpapi/realtime_socket.go` | WebSocket 入口，接收 `click` 消息 |
| `internal/httpapi/realtime_helpers.go` | `executeButtonClick()` 统一点击处理 |
| `internal/httpapi/afk_service.go` | 挂机自动攻击服务 |
| `internal/httpapi/equipment_routes.go` | 装备操作 HTTP API（穿戴/卸下/强化/分解） |
| `internal/vote/store.go` | 核心业务逻辑：伤害计算、击杀结算、掉落掷骰、物品生成 |
| `internal/vote/rarity.go` | 稀有度定义、掉落权重倍率、属性范围随机生成 |
| `internal/vote/boss_pool.go` | Boss 模板池、循环队列、掉落池设置 |
| `internal/vote/admin.go` | 后台管理：装备模板 CRUD、Boss 掉落池覆盖 |
| `internal/vote/public_resources.go` | 公共资源接口（Boss 掉落池展示给前端） |

---

## 一、入口层

### 1.1 手动点击（WebSocket）

```
前端 → WebSocket /api/ws → { type: "click", slug: "boss-part:X-Y" }
```

调用链：

- `realtimeSession.handleMessage()` (realtime_socket.go)
- → `executeButtonClick()` (realtime_helpers.go)
- → `resolveClickNickname()` — 解析昵称
- → `enforceClickRateLimitForClient()` — IP+昵称限流
- → `Store.ClickButton()` (store.go)
- → `Store.clickBossPart()` (store.go)

### 1.2 挂机自动攻击

```
AfkService.loop() 每秒 tick → runPlayerOnce() → Store.AttackBossPartAFK()
```

调用链：

- `AfkService.loop()` (afk_service.go) — 每秒 tick
- → `AfkService.runPlayerOnce()` (afk_service.go)
- → 判断玩家离开页面超过 60 秒 (`afkStartDelay`)
- → `Store.AttackBossPartAFK()` (store.go)

## 二、伤害计算

### 2.1 手动点击路径（完整公式）

1. **加载战斗属性** — `combatStatsForNickname()` (store.go)
   - 基础攻击力 5 + 穿戴装备的 `AttackPower` 累加
   - 装备的 `ArmorPenPercent`、`CritRate`、`CritDamageMultiplier` 累加
   - 叠加天赋修正（`ComputeTalentModifiers`）

2. **暴击判定** — `nextIncrement()` (store.go)
   - 通过 `criticalRollPlan()` 将暴击率转为整数掷骰方案
   - 暴击时使用 `CriticalDamage`（取计数法和倍率法中较大值）

3. **部位伤害公式** — `CalcBossPartDamage()` (store.go)
   ```
   有效护甲 = partArmor * (1 - 破甲率)    // 上限 80% 减免
   基础伤害 = max(攻击力 * 部位系数 - 有效护甲, 1)
   增伤乘区 = 1 + 全局增伤 + 部位类型增伤 + 每存活部位增伤 + 低血斩杀增伤
   最终伤害 = 基础伤害 * 增伤乘区 * 暴击倍率
   ```
   - 部位系数：软组织 1.0、重甲 0.4、弱点 2.5

4. **天赋触发追击** — `applyTriggeredTalentDamage()` (store.go)
   - 天赋 `normal_core`：每 200 次点击触发一次追击伤害
   - 追击伤害 = 基础伤害 * chaseRatio * extraHits

### 2.2 挂机攻击路径（简化版）

`AttackBossPartAFK()` (store.go)：
- 伤害 = `floor(攻击力 * 0.5)`
- 不增加点击数，不触发暴击

---

## 三、Boss 击杀判定

在 `applyBossPartDamage()` (store.go) 或 `AttackBossPartAFK()` (store.go) 中：

- 遍历所有部位，检查是否全部 HP=0
- 若 `allDead = true` → 设置 `boss.Status = bossStatusDefeated`
- 记录 `boss.DefeatedAt` 时间戳
- 进入 `finalizeBossKill()` 结算

---

## 四、核心结算：`finalizeBossKill()`

**位置**：store.go

这是装备掉落的核心函数，完整流程分 6 步：

### 4.1 防重复结算

- Redis `SetNX` 分布式锁，key 为 `boss:{bossID}:reward:lock`
- 确保同一 Boss 只结算一次

### 4.2 加载掉落池

- `loadBossLoot()` (store.go)
- 从 Redis ZSet `{namespace}boss:{bossID}:loot` 读取
- 每个条目的 score 即 `DropRatePercent`（0~100）
- 关联装备模板的完整属性（名称、图标、攻击力等）

### 4.3 获取参与者列表

- 从伤害排行榜（Redis ZSet）获取所有参与者及其总伤害
- 最低伤害门槛 = `ceil(boss.MaxHP / 100)`（至少造成 1% 总血量才算参与）

### 4.4 资源奖励

- `rollResourceReward()` (store.go)
  - 金币：base * [0.75, 1.25] 随机
  - 强化石：base * [0.67, 1.33] 随机
- 挂机模式（`afkMode=true`）金币和强化石基础值减半

### 4.5 装备掉落掷骰

- `rollLootDrops()` (store.go)
- 对掉落池中每个条目**独立概率掷骰**（非互斥抽取）
- 10000 面骰，`< threshold` 则掉落
- `threshold = DropRatePercent * 100`
- 多个条目可以同时掉落

### 4.6 物品实例生成

- `newEquipmentInstanceID()` — Redis INCR 生成 `inst-1, inst-2, ...`
- 创建装备实例 Hash（Redis key: `{namespace}equip:inst:{id}`）
- 将实例 ID 加入玩家的实例集合
- 记录 `lastReward`（最近一次掉落的完整奖励列表 JSON）
- `SaveBossToHistory()` — 将被击败的 Boss 存入历史
- 若开启 Boss 循环 → `activateNextBossFromCycle()` 自动激活下一个 Boss

---

## 五、掉落概率机制

**关键特性**：每个掉落条目**独立概率掷骰**，不是互斥抽取。

- `DropRatePercent=10` → 10% 掉落概率
- `DropRatePercent=100` → 100% 必掉
- 多个条目可以同时掉落

## 六、前端返回方式

### 6.1 手动点击 → WebSocket `click_ack`

`click_ack` 消息包含：

```json
{
  "type": "click_ack",
  "payload": {
    "delta": 5,
    "critical": false,
    "bossDamage": 12,
    "damageType": "normal",
    "talentEvents": [],
    "partStateDeltas": [{ "x": 0, "y": 0, "damage": 12, "beforeHp": 88, "afterHp": 76 }],
    "userDelta": { "gold": 1500, "stones": 30, "talentPoints": 5 },
    "button": { "key": "boss-part:0-0" }
  }
}
```

装备奖励不通过 `click_ack` 返回，而是在下次 `sync_request` 或 `hello` 时通过 `snapshot.user.recentRewards` 字段获取。

当 Boss 被击杀时，`BroadcastUserAll=true`，触发全局广播，所有客户端通过 `public_delta` 收到新 Boss 状态。

### 6.2 挂机攻击 → 领取结算

1. 挂机期间 `AfkService` 每秒攻击一次
2. 每次击杀，奖励累积到 `afkPlayerState.Rewards`
3. 玩家返回页面 → `stopAfk()` → 合并结算写入 Redis
4. 前端调用 `GET /api/player/afk/settlement` → `ConsumeSettlement()` 返回：

```json
{
  "kills": 3,
  "goldTotal": 4500,
  "stoneTotal": 90,
  "startedAt": 1713000000,
  "endedAt": 1713003600,
  "rewards": [
    { "bossId": "boss-1", "bossName": "裂隙领主", "itemId": "fire-sword", "itemName": "烈焰之剑", "grantedAt": 1713001200 }
  ]
}
```

### 6.3 掉落池展示

前端通过 `GET /api/boss-resources`（`GetBossResources()`）获取当前 Boss 的完整掉落池：

```json
{
  "bossId": "raid-1",
  "goldRange": { "min": 750, "max": 1250 },
  "stoneRange": { "min": 67, "max": 133 },
  "talentPointsOnKill": 5,
  "bossLoot": [
    {
      "itemId": "fire-ring",
      "itemName": "烈焰戒指",
      "slot": "accessory",
      "rarity": "史诗",
      "dropRatePercent": 30,
      "attackPower": 18,
      "critRate": 0.22,
      "imagePath": "https://cdn.example.com/items/fire-ring.png"
    }
  ]
}
```

## 七、掉落池配置

掉落池由管理员通过后台配置，有两级：

1. **Boss 模板掉落池** (`SetBossTemplateLoot`)
   - Redis key: `{namespace}boss-template:{templateID}:loot`
   - 模板被激活时，掉落池复制到实例级

2. **Boss 实例掉落池** (`SetBossLoot`)
   - Redis key: `{namespace}boss:{bossID}:loot`
   - 可覆盖当前 Boss 的掉落池

掉落池存储为 Redis ZSet，member = itemID，score = DropRatePercent。

## 八、稀有度系统

**文件**：rarity.go

| 稀有度 | 攻击力范围 | 掉落权重倍率 | 分解原石 |
|---|---|---|---|
| 普通 | 1~2 | 1.0 | 1 |
| 优秀 | 2~4 | 0.8 | 2 |
| 稀有 | 4~8 | 0.5 | 5 |
| 史诗 | 8~15 | 0.3 | 10 |
| 传说 | 15~25 | 0.15 | 20 |
| 至臻 | 25~40 | 0.05 | 50 |

`GenerateRarityStats()` 用于根据稀有度随机生成装备属性值（攻击、破甲、暴击伤害、Boss 增伤）。

## 十、装备后续操作

装备掉落生成后，玩家可进行以下操作（全部在 equipment_routes.go 中注册）：

| 操作 | 端点 | 后端函数 |
|---|---|---|
| 穿戴 | `POST /api/equipment/:instanceId/equip` | `Store.EquipItem()` |
| 卸下 | `POST /api/equipment/:instanceId/unequip` | `Store.UnequipItem()` |
| 强化 | `POST /api/equipment/:instanceId/enhance` | `Store.EnhanceItem()` |
| 锁定 | `POST /api/equipment/:instanceId/lock` | `Store.LockItem()` |
| 解锁 | `POST /api/equipment/:instanceId/unlock` | `Store.UnlockItem()` |
| 分解 | `POST /api/equipment/:instanceId/salvage` | `Store.SalvageItem()` |
| 一键分解 | `POST /api/equipment/salvage/unequipped` | `Store.BulkSalvageUnequipped()` |

穿戴装备会直接影响战斗属性（`loadoutBonuses()`），从而提升后续攻击的伤害和暴击能力。
