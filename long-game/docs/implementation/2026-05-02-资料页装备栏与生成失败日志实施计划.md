# 资料页装备栏与生成失败日志实施计划

## 目标

本轮只处理三件事：

1. 资料页战斗属性改成单张紧凑摘要卡，一行两个属性，不再拆分基础属性与强化属性。
2. 资料页装备栏改成左右 `3 + 3` 的正方形槽位布局，空槽显示部位名，已装备槽位显示装备图片，并复用背包装备详情浮层。
3. 后台管理生成装备草稿失败时，将失败上下文写入 Mongo，便于后续分析与调整生成算法。

## 设计决策

### 战斗属性

- 只显示对玩家有解释力的当前战斗属性。
- 所有数值直接取 `combatStats`，即装备与天赋已合并后的最终值。
- 百分比类最多显示两位小数。
- `护甲穿透` 显示真实百分比，`0.01` 显示 `1%`。
- `暴击倍率` 显示成百分比增量，例如 `1.5` 显示 `+50%`。

### 装备栏

- 布局固定为两列，每列三个槽位。
- 每个槽位主视觉为正方形框。
- 已装备时优先显示 `imagePath` 对应图片。
- 未装备时显示部位名占位。
- 悬浮提示复用背包现有装备属性浮层结构，不额外引入新的交互体系。

### 失败日志

- 新增 Mongo 集合 `equipment_draft_failures`。
- 失败记录至少包含：
  - `prompt`
  - `draft`
  - `error_message`
  - `raw_response`
  - `created_at`
- 仅在生成失败时写入；成功生成不写失败集合。
- 生成器会尽量把失败时可获得的草稿与模型原始返回内容挂在错误对象上，供路由层统一记录。

## 验证

### 前端

- 静态测试确认：
  - 不再出现基础属性 / 强化属性双卡片
  - 采用紧凑属性列表
  - 装备栏为 `3 + 3` 正方形槽位
  - 已装备槽位存在图片与 tooltip 结构

### 后端

- 路由测试确认失败时会调用失败日志写入器，并记录提示词、错误、草稿与模型原始返回。
- 生成器测试确认失败错误能携带原始返回内容。
