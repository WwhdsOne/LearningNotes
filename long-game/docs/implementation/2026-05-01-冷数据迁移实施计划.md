# 冷数据迁移实施计划

## 目标

围绕 Mongo 主存方案，完成以下工作：

- 将 Boss 历史切换到 Mongo
- 将留言墙历史切换到 Mongo
- 将后台操作审计日志接入 Mongo
- 将系统日志与业务事件日志接入 Mongo
- 提供正式服可用的停机迁移工具
- 迁移验证通过后删除 Redis 冷数据

## 实施顺序

## 阶段 1：补齐 Mongo 仓储

### 任务 1.1：留言墙 Mongo 仓储

新增：

- `backend/internal/mongostore/messages.go`

内容：

- `CreateMessage`
- `ListMessages`
- `DeleteMessage`
- 索引初始化

验证：

- 新增 Mongo 仓储单元测试
- 保证分页顺序与现有接口兼容

### 任务 1.2：审计日志 Mongo 仓储

新增：

- `backend/internal/mongostore/admin_audit.go`

内容：

- `WriteAdminAuditLog`

验证：

- 能落库
- 能按操作者和时间检索

### 任务 1.3：事件与系统日志 Mongo 仓储

新增：

- `backend/internal/mongostore/domain_events.go`
- `backend/internal/mongostore/system_logs.go`

内容：

- `WriteDomainEvent`
- `WriteSystemLog`

验证：

- 能写入基本事件结构

## 阶段 2：迁移工具

### 任务 2.1：工具入口

新增：

- `backend/cmd/migratecolddata/main.go`

子命令：

- `plan`
- `migrate`
- `verify`
- `cleanup`

验证：

- 命令行能正确解析参数
- 能读取 Consul 配置

### 任务 2.2：Boss 历史迁移

复用现有 Redis 读取逻辑：

- `ListBossHistory`

迁入：

- `boss_history`

要求：

- upsert 幂等
- 打印迁移数量

验证：

- 迁前迁后数量一致

### 任务 2.3：留言墙历史迁移

复用现有 Redis 读取逻辑：

- `ListMessages`

迁入：

- `wall_messages`

要求：

- 全量翻页读取
- upsert 幂等

验证：

- 迁前迁后数量一致

### 任务 2.4：verify 校验

内容：

- 核对 Boss 历史数量
- 核对留言数量
- 抽样比较关键字段

验证：

- 失败时返回非零退出码

### 任务 2.5：cleanup 删除 Redis 冷数据

内容：

- 删除 Boss 历史相关 key
- 删除留言墙历史相关 key

要求：

- 不删除任何实时 key
- 执行前打印删除清单摘要

验证：

- 删除后 Redis 不再保留冷数据 key

## 阶段 3：应用层切换到 Mongo 主存

### 任务 3.1：Boss 历史固定读写 Mongo

改动：

- 去掉 Boss 历史 Redis 主写依赖
- 后台 Boss 历史分页固定读 Mongo
- 公共 Boss 历史接口固定读 Mongo

验证：

- HTTP 测试通过
- 无 Redis/Mongo 读源选择开关残留

### 任务 3.2：留言墙固定读写 Mongo

改动：

- 创建留言写 Mongo
- 留言分页读 Mongo
- 删除留言删 Mongo

验证：

- 现有留言接口测试通过
- 翻页行为兼容

## 阶段 4：审计日志与事件日志接入

### 任务 4.1：后台审计日志

接入点优先：

- 管理员登录
- 修改 Boss 模板
- 启停 Boss 循环
- 重置玩家密码
- 删除留言

验证：

- 关键后台写操作有审计记录

### 任务 4.2：业务事件日志

接入点优先：

- Boss 激活
- Boss 击杀
- 掉落结算
- 装备生成/重铸/分解

验证：

- 关键事件会写入 `domain_events`

### 任务 4.3：系统日志落 Mongo

接入策略：

- `warn`
- `error`
- 高价值异常

验证：

- 不做全量请求日志落库
- 只落高价值日志

## 阶段 5：测试服演练

执行顺序：

1. 停服务
2. `plan`
3. `migrate`
4. `verify`
5. `cleanup`
6. 启动服务
7. 回归 Boss 历史与留言功能

## 验收标准

- Boss 历史完全从 Mongo 读写
- 留言墙历史完全从 Mongo 读写
- Redis 不再存这两类冷数据
- 审计日志和事件日志开始写 Mongo
- 迁移工具可在正式服重复执行
- 全量后端测试通过
