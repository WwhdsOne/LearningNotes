# Boss 战斗区右侧信息与连击跳字布局调整

## 背景

原先以下信息展示在 5x5 Boss 战斗区下方：

- 我的伤害
- Boss 榜人数
- 1% 掉落资格说明
- Boss 掉落池入口
- 掉落物数量

这会拉长战斗区下方内容，挤占主战场阅读空间。与此同时，连击加成跳字原本占据连击框下一行，视觉上与主数字分离，显得松散。

## 本次调整

### 右侧信息整合

将上述 Boss 信息整体并入 5x5 战斗区右侧下方，新增 `boss-right-summary` 信息卡，位于：

- 部位系数
- 伤害类型
- 战报/掉落信息卡

信息卡结构：

1. 第一行：`我的伤害`、`Boss 榜`
2. 第二行：`至少造成 1% Boss 生命值才有掉落资格`
3. 第三行：`点击查看 Boss 掉落池` + `掉落物数量`

同时移除战斗区下方原有的 `vote-stage__boss-hud-stats` 与 `vote-stage__boss-note` 模板块，避免重复展示。

### 连击跳字位置

保留“每 25 连击触发一次跳字”的逻辑，但将跳字从连击框下一行移动到 `连击 xN` 数字的右上角。

表现形式：

- 紧贴主数字右上角出现
- 轻微放大后向右上漂移
- 最后淡出

这样可以让提示更像附着在连击主反馈上的奖励信息，而不是独立占据一行说明文。
