# 伤害动画解析优先级

本文总结当前前端 Boss 战斗页里，伤害浮字与舞台特效的解析顺序，便于后续调整动画或补充新伤害类型时对照。

## 相关入口

- `frontend/src/pages/publicPageState.js`
- `frontend/src/pages/BattlePage.vue`

其中：

- `resolveDamageVariant()` 负责把一次命中归类成前端使用的伤害类型
- `triggerDamageBurst()` 负责生成浮字、粒子，并触发舞台级特效
- `DAMAGE_PRIORITY` 负责同一部位同时存在多条伤害浮字时的显示排序

## 一、显示优先级

当前前端源码中的 `DAMAGE_PRIORITY` 为：

`doomsday -> judgement -> weakCritical -> critical -> trueDamage -> pursuit -> heavy -> normal`

含义如下：

- `doomsday`：末日类削血伤害，优先级最高
- `judgement`：终结类伤害
- `weakCritical`：弱点暴击
- `critical`：普通暴击
- `trueDamage`：真实伤害
- `pursuit`：追击伤害
- `heavy`：普通命中重甲时的灰色伤害表现
- `normal`：普通伤害

`triggerDamageBurst()` 会把同一格子的浮字按这个顺序排序，数值越靠前越先显示，并且最多保留 6 条。

## 二、单次命中的类型解析顺序

`resolveDamageVariant()` 的判断顺序如下：

1. 先看服务端是否显式下发 `damageType` / `hitType` / `effectType`
2. 没显式类型时，如果本次命中是暴击且目标部位是 `weak`，归类为 `weakCritical`
3. 否则如果是暴击，归类为 `critical`
4. 否则如果该部位当前处于 `崩塌` 状态，归类为 `trueDamage`
5. 否则如果目标部位是 `heavy`，归类为 `heavy`
6. 否则如果来源是 `auto` 或 `afk`，归类为 `pursuit`
7. 以上都不满足时，归类为 `normal`

也就是说，前端本地推导时，“弱点暴击”会盖过“普通暴击”，普通命中重甲会落到 `heavy` 分支；但如果该重甲格子已经触发崩塌，则提升为紫色 `⚡` 真实伤害特效。

## 三、动画和舞台特效结算位置

`triggerDamageBurst()` 在确定伤害类型后，会读取 `DAMAGE_VARIANTS`：

- 生成浮字文案、缩放倍率、存在时长
- 生成粒子数量和颜色
- 依次触发该类型绑定的 `stageFx`
- 如果该类型额外带 `shake` 时长，再单独触发震屏

当前几个关键类型对应关系：

- `normal`：普通浮字，无舞台特效
- `pursuit`：附带 `flash`
- `heavy`：灰色小号浮字，无舞台特效
- `trueDamage`：紫色 `⚡` 浮字，常见于已崩塌的重甲格子
- `critical`：附带 `shake`
- `weakCritical`：附带 `shake + flash`
- `doomsday`：附带 `shake + doom + blade`
- `judgement`：附带 `shake + slowMo + vignette`

## 四、BattlePage 上的实际呈现

`BattlePage.vue` 通过 `damageStageFx` 把舞台特效映射到页面类名：

- `damage-stage--shake`
- `damage-stage--flash`
- `damage-stage--doom`
- `damage-stage--blade`
- `damage-stage--slowmo`
- `damage-stage--vignette`

伤害类型说明区当前展示为：

- 普通
- 暴击 `CRIT!`
- 弱点暴击 `WEAK!`
- 追击
- `⚡️`真实伤害
- `💀` 削血
- `K.O.` 终结

## 六、当前关键阈值

- `doomsday`：单次伤害达到 Boss 最大生命值 `20%`
- `judgement`：暴击且单次伤害达到 Boss 最大生命值 `10%`
- `trueDamage`：命中已崩塌的重甲格子，或后端显式下发 `damageType=trueDamage`

## 五、后续修改建议

- 如果后端要强制指定动画类型，优先直接下发 `damageType`
- 如果要新增一种更高规格的终结动画，需要同时更新 `normalizeDamageVariant()`、`DAMAGE_VARIANTS`、`DAMAGE_PRIORITY` 和 `BattlePage.vue` 右侧说明
- 如果只是改文案，不需要动 `resolveDamageVariant()`，只改 `DAMAGE_VARIANTS` 的 `label` 和战斗页说明即可
