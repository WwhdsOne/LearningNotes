# 任务红点轮询与资料页稀有度样式实施计划

## 目标

本轮处理两件事：

1. 让前台顶部“任务”红点在登录后即可初始化，并在玩家持续点击过程中按固定频率同步更新。
2. 让资料页背包栏和装备栏的装备稀有度展示复用 Boss 掉落池现有的颜色样式。

## 设计

### 任务红点

- 登录成功后立即拉一次 `/api/player/profile`，保证首次任务列表和红点状态正确。
- 已登录期间每隔 10 秒拉一次 `/api/tasks`。
- 轮询只更新 `tasks`，不重复拉整份 profile，避免无关资源状态频繁刷新。
- 登出、页面卸载、未登录时停止轮询。

### 稀有度样式

- 继续复用 `getRarityClassName()` 返回的稀有度 class。
- 资料页装备栏中“稀有度 + 装备名”整体按稀有度着色。
- 资料页背包栏的装备名与稀有度标签都按同一套稀有度样式显示。
- 不新增新的稀有度视觉体系，直接跟 Boss 掉落池保持一致。

- 跑全量前端测试确保不影响现有页面拆分与任务逻辑。
