# Boss 部位点击扣血持久化修复

## 问题

点击 Boss 分区后，前端看不到 Boss 总血量和部位血量按预期减少。

## 根因

部位点击分支只把 `parts` 写回 Redis，没有把 `current_hp`、`status` 和 `defeated_at` 一起写回。

同时，原逻辑用理论伤害扣总血量；当伤害超过部位剩余血量时，可能再次造成 Boss 总血量和部位剩余血量之和不一致。

## 修复

- 部位点击后按部位实际损失血量计入 Boss 伤害。
- Boss 总血量改为由所有部位当前血量合计得出。
- 点击后同时持久化 `parts`、`current_hp`、`status`、`defeated_at`。
- 最后一块部位死亡时，持久化 `defeated` 状态后再进入击杀结算。
