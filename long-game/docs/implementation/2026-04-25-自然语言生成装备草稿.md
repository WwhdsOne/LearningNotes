# 自然语言生成装备草稿

## 目标

后台装备页新增“自然语言生成装备”能力。管理员输入装备描述后，后端调用 OpenAI Chat Completions 兼容接口生成严格 JSON 装备草稿；前端只填充表单预览，不自动保存。管理员确认后仍走现有装备保存接口。

## 落地范围

- 后端新增 `llm` 配置，默认 `base_url` 为 `https://api.openai.com/v1`。
- 后端新增 `POST /api/admin/equipment/generate`，需要管理员登录。
- 后端新增 OpenAI 兼容 Chat Completions 客户端，使用 `response_format.type=json_schema` 和本地严格校验。
- 装备草稿只允许现有字段，拒绝攻速、点击间隔、额外点击概率等旧词缀字段。
- 修正装备模板列表读取链路，确保 `imagePath`、`imageAlt`、`talentAffinity` 可读回。
- 前端装备页新增“新增装备”按钮、自然语言生成区、左右标签式表单和 5 列响应式装备卡片网格。

## 校验

- `rtk go test ./...`
- `rtk npm --prefix frontend run test`
- `rtk npm --prefix frontend run build`
