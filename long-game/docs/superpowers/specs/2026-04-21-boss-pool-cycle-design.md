# Boss 池循环设计

**Goal:** 在不改动前台玩家视角的前提下，为后台增加可维护的 Boss 池与模板掉落池，并支持 Boss 被击败后立即随机切下一只，保证现场持续有 Boss 可打。

## 需求

- `/admin` 可维护多只 Boss 模板，每只模板包含：
  - `templateId`
  - `name`
  - `maxHp`
  - 独立掉落池
- 后台可开启或关闭 Boss 循环。
- 循环开启时：
  - 如果当前没有活动 Boss，立即随机刷出一只。
  - 当前 Boss 被击败后，立即随机刷出下一只。
  - 允许连续抽到同一模板，不做去重。
- 前台玩家仍然只看到当前这一只 Boss 与它当前掉落池，不暴露“Boss 池”概念。
- 历史 Boss、伤害榜、掉落记录不能因为模板重复出现而串数据。

## 数据模型

- 当前 Boss 继续使用 `vote:boss:current`，但改为存储“实例 Boss”。
- 新增 Boss 模板池：
  - `vote:boss:pool:index`
  - `vote:boss:pool:<templateId>`
  - `vote:boss:pool:<templateId>:loot`
- 新增循环状态：
  - `vote:boss:cycle`
- 当前刷出的实例 Boss 继续使用已有实例级数据：
  - `vote:boss:<instanceId>:damage`
  - `vote:boss:<instanceId>:loot`
- 实例 Boss 支持记录 `templateId`，用于后台识别来源；前台可忽略该字段。

## 运行规则

- 开启循环时：
  - 若存在活动 Boss，则只打开循环开关。
  - 若没有活动 Boss 或当前 Boss 已失效，则立刻从池中随机抽一只模板，生成唯一实例并设置为当前 Boss。
- Boss 被击败时：
  - 先完成伤害结算、掉落发放。
  - 将被击败的实例写入历史。
  - 如果循环开启且池非空，立刻随机生成下一只实例 Boss 覆盖当前 Boss。
- 手动关闭当前 Boss 时：
  - 若循环关闭，则清空当前 Boss。
  - 若循环开启，则视为“跳过当前 Boss”，立刻补下一只。
- 编辑模板掉落池只影响未来刷出的实例，不回写当前实例与历史实例的掉落快照。

## 后台交互

- Boss 页新增三块信息：
  - 当前循环状态与当前 Boss 状态
  - Boss 模板池管理
  - 模板掉落池编辑
- 当前 Boss 掉落池只作为展示结果，真正可编辑的是模板掉落池。
- 删除模板不会影响已经刷出的实例 Boss，只影响未来抽取范围。

## 测试重点

- 循环开启且无 Boss 时，立即刷出实例 Boss。
- Boss 被击败后，会保存历史并自动切到下一只。
- 连续两次刷到同一模板是合法行为。
- 模板掉落池修改后，不影响已经在场或已进入历史的实例掉落快照。
- 关闭循环后，Boss 被击败不再自动补位。
