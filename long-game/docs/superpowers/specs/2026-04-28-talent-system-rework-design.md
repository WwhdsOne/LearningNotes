# 天赋系统重设计

> 状态：当前正式口径。
>
> 当前三系天赋的结构、数值、成本模型、总成本测算均以本文为准。

日期：2026-04-28

## 一、目标

1. 三系天赋树全部改为等级制节点（Lv1~Lv5），数值大幅提高
2. 暴击系重构：死兆收割改为被动、死亡狂喜改为终极、末日审判改为小技能
3. 普攻系和破甲系保留机制，调高数值、降低触发门槛
4. 若维持原天赋点产出速度，Lv1 全部解锁约 12-17 天，Lv5 全部学满约 4.8-6.4 个月

## 二、升级成本

### 基准成本

| 层级 | 主节点 Lv1 | 小节点 Lv1 |
|---|---|---|
| Tier0 基石 | 20 | — |
| Tier1 | 30 | 15 |
| Tier2 | 80 | 35 |
| Tier3 | 150 | 60 |
| Tier4 终极 | 200 | — |

### 升级公式

```
cost(level) = round(baseCost × level^0.85 × 1.8)
```

说明：

- `cost(level)` 表示“升到该等级这一次”需要支付的天赋点，不是累计值
- 从 LvA 升到 LvB 的实际消耗 = `sum(cost(level))`，其中 `level ∈ [A+1, B]`
- 洗点返还同样按已支付的逐级成本累计返还

Tier0 基石符文单独使用更陡的公式：

```
cost(level) = round(baseCost × 1.8 × 3^(level - 1))
```

说明：

- 仅 `Tier0 基石` 使用该公式
- 其他 Tier 与小节点仍保持 `round(baseCost × level^0.85 × 1.8)`
- 因此基石从 Lv2 开始会显著抬升，洗点返还也按这条公式的实际已支付成本返还

## 三、等级制规则

- 每个节点最高 Lv5
- 升级时计算逐级差价：`sum(cost(level))`，支付差额即可
- 前置和层锁只检查 Lv1 是否已学
- 效果叠加：`倍率 × 等级`（如 Lv3 残忍 = 120% × 3 = 360% 暴伤）
- 前端展示：节点图标带等级角标，点击可升级

### 末日审判（新小技能）

| 等级 | 标记数 | 每标记获层 | 总层数上限 | Lv1 成本 |
|---|---|---|---|---|
| Lv1 | 2 | 25 | 50 | 30 |
| Lv2 | 4 | 25 | 100 | |
| Lv3 | 7 | 30 | 210 | |
| Lv4 | 10 | 35 | 350 | |
| Lv5 | 15 | 40 | 600 | |

- 触发条件：被标记部位累计被打掉 30% 最大血量时触发（不需击碎，可多次触发）
- 标记随机分配存活部位
- 同一部位被多次标记时，每次独立追踪 30% 血量进度

### 死亡狂喜（新终极技能）

- 触发条件：死兆层数 ≥ 100
- 消耗：固定 100 层（多余保留）
- 伤害公式：`baseDamage × min(消耗层数, 100) × 暴伤倍率`
- 层数系数上限锁死 100

| 等级 | 暴伤倍率 |
|---|---|
| Lv1 | ×1.0 |
| Lv2 | ×1.5 |
| Lv3 | ×2.0 |
| Lv4 | ×2.5 |
| Lv5 | ×3.0 |

### 死兆收割（改为被动）

| 层数档位 | 增伤 |
|---|---|
| ≥30 | `baseDamage × 1.5` |
| ≥60 | `baseDamage × 2.0` |
| ≥90 | `baseDamage × 2.5` |
| ≥120 | `baseDamage × 3.0` |

- 不消耗层数，达到即生效
- 与其他被动（斩杀预兆、死兆共鸣）乘法叠加

## 九、节点视觉等级表现

节点从"未点"到 Lv5 逐级提升亮度和粒子效果，让玩家直观感受到投入深度。

| 等级 | 亮度 | 光晕 (box-shadow) | 粒子效果 | 边框 |
|---|---|---|---|---|
| 未学 | 0.4 opacity | 无 | 无 | 灰色虚线 |
| Lv1 | 1.0 opacity | 微弱同色辉光，模糊半径 4px | 无 | 实色细线 |
| Lv2 | 1.0 + brightness(1.1) | 辉光半径 8px | 零星 2-3 粒子缓慢漂浮 | 边框微亮 |
| Lv3 | 1.0 + brightness(1.2) | 辉光半径 14px，外扩 | 6-8 粒子环绕 | 边框发光 |
| Lv4 | 1.0 + brightness(1.35) | 辉光半径 22px，双层 | 12-15 粒子活跃旋转 | 亮色粗边框 |
| Lv5 | 1.0 + brightness(1.5) | 辉光半径 30px，脉冲动画 | 20+ 粒子高速旋转 + 间歇爆发 | 金色/彩色流光边框 |

**粒子系统参数**：
- 粒子大小：2px~4px 随机
- 颜色：取节点主色，随机 ±15% 色调偏移
- Lv3-4 慢速环绕，Lv5 高速旋转 + 每 3~5 秒一次粒子爆发（向外扩散后回收）

**亮度与辉光的 CSS 实现方向**：

- `filter: brightness()` 控制核心亮度
- `box-shadow` 多层叠加实现光晕（内层扩散小亮度高，外层扩散大亮度低）
- Lv5 脉冲动画 `@keyframes`：光晕半径在 28px~34px 之间正弦波动，周期 2 秒

**三系颜色参考**：
- 普攻（均衡攻势）：琥珀/金 `#f0a030`
- 破甲（碎盾攻坚）：冰蓝 `#4090e0`
- 暴击（致命洞察）：赤红 `#e04040`

## 十、实施要点

1. `TalentDef` 增加 `MaxLevel int` 字段，默认 5
2. `TalentState` 存储改为 `map[string]int`（talentID → 当前等级）
3. `LearnTalent` 改为 `UpgradeTalent`，支持检测当前等级、计算差价
4. `ComputeTalentModifiers` 按等级累加倍率
5. `applyTriggeredTalentDamage` 中：
   - 删除死兆收割的消耗逻辑，改为在伤害计算时查询当前层数对应的被动档位
   - 删除死亡狂喜的 duration-based 逻辑，改为层数 ≥100 时立即触发
   - 末日审判的标记触发逻辑：追踪每个被标记部位累计受到的伤害，达到 30% 最大血量时触发
6. 前端 `TalentTree.vue` 等组件需适配：
   - 等级制展示（节点图标带等级数字角标）
   - 升级交互（点击已学节点弹出升级确认）
   - 第九节的视觉等级效果（亮度递进 + 粒子系统）
7. `talentDefs` 中 `crit_ultimate` 的 `EffectType` 从 `doom_judgment` 改为新类型
8. `talentDefs` 中新增末日审判的定义，移除旧的 `doom_judgment` 定义
