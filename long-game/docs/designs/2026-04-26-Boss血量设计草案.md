## 一、核心规则

| 规则                 | 说明                                      |
| :------------------- | :---------------------------------------- |
| 同时只有一个Boss     | 全服所有玩家面对同一个Boss                |
| 全服共享血量         | 所有玩家点击同一个部位、同一个总血量      |
| Boss血量动态         | 基础血量 × 在线人数系数                   |
| 系数在Boss生成时计算 | 该Boss存活期间不再变化                    |
| Boss死亡自动切换     | 按管理员配置的循环队列逐个推进            |
| 单人点击基础伤害     | 每次点击基础5点，装备攻击力按伤害公式叠加 |

## 二、各阶段攻击力基准

### 2.1 全身攻击力总和

玩家全身6件装备（武器、头盔、胸甲、手套、腿甲、饰品），各部位攻击力按稀有度取中位值，+5强化后汇总。

| Boss阶段  | 对应稀有度 | 全身攻击力中位值 | +5强化后 | 软组织每击伤害 |
| :-------- | :--------- | :--------------- | :------- | :------------- |
| 新手Boss  | 普通       | 30               | 38       | 38             |
| 常规Boss  | 稀有       | 100              | 128      | 128            |
| 高阶Boss  | 史诗       | 200              | 255      | 255            |
| Raid Boss | 传说       | 400              | 510      | 510            |
| 终极Boss  | 神话~至臻  | 800              | 1,020    | 1,020          |

强化公式：1.05^5 ≈ 1.276，取1.28方便计算。

### 2.2 各部位攻击力参考（普通稀有度）

| 部位 | 攻击力范围 | 中位值 |
| :--- | :--------- | :----- |
| 武器 | 5~30       | 17     |
| 手套 | 3~21       | 12     |
| 头盔 | 2~14       | 8      |
| 胸甲 | 1~11       | 6      |
| 腿甲 | 2~14       | 8      |
| 饰品 | 1~8        | 4      |

全身中位值总和约55，但受装备品质和部位修正影响，实际约为30。

### 2.3 伤害公式

text

```
伤害 = MAX((攻击力 × 部位系数 - 有效护甲), 1) × (1 + 全伤害增幅) × 暴击乘区
```



部位系数：软组织1.0 / 重甲0.4 / 弱点2.5

## 三、动态血量机制

Boss总血量 = 基础血量 × 在线人数系数

| 在线人数  | 系数 | 设计意图                   |
| :-------- | :--- | :------------------------- |
| 1~5人     | 1.0  | 凌晨时段，单人也能慢慢推进 |
| 6~20人    | 1.5  | 有人帮忙，但不会秒杀       |
| 21~50人   | 2.5  | 明显加速，协作感体现       |
| 51~100人  | 4.0  | 快速推进，热闹             |
| 100人以上 | 6.0  | 百人团战，Boss蒸发         |

系数在Boss生成时计算一次，存活期间不变。

------

## 四、Boss血量设计

### 4.1 基础血量（系数1.0时）

目标单人击杀时间 × 软组织每击伤害 × 每秒点击5~8次 × 血量倍率1.2。

| Boss阶段  | 目标单人击杀时间 | 软组织每击伤害 | 单人总伤害估算      | 基础总血量         |
| :-------- | :--------------- | :------------- | :------------------ | :----------------- |
| 新手Boss  | 1~3分钟          | 38             | 11,400~54,720       | **35,000**         |
| 常规Boss  | 5~15分钟         | 128            | 192,000~921,600     | **500,000**        |
| 高阶Boss  | 15~40分钟        | 255            | 1,147,500~4,896,000 | **2,500,000**      |
| Raid Boss | 单人不可能       | 510            | —                   | **250,000,000**    |
| 终极Boss  | 单人不可能       | 1,020          | —                   | **25,000,000,000** |

### 4.2 动态血量示例

新手Boss基础血量35,000：

| 在线人数  | 系数 | 实际总血量 | 单人预期击杀时间 |
| :-------- | :--- | :--------- | :--------------- |
| 1~5人     | 1.0  | 35,000     | 1~3分钟          |
| 6~20人    | 1.5  | 52,500     | 单人约2~5分钟    |
| 21~50人   | 2.5  | 87,500     | 单人约3~8分钟    |
| 51~100人  | 4.0  | 140,000    | 百人团几十秒     |
| 100人以上 | 6.0  | 210,000    | 百人团不到一分钟 |

## 五、部位血量分配

### 5.1 分配规则

部位血量占比按类型差异化：

| 部位类型 | 血量占比 | 说明                           |
| :------- | :------- | :----------------------------- |
| 软组织   | 25%~35%  | 血量适中，主要输出目标         |
| 重甲     | 35%~50%  | 血量厚，加上高护甲需要更多伤害 |
| 弱点     | 10%~15%  | 血量少，优先集火目标           |

### 5.2 分配示例

新手Boss，3部位，总血量35,000：

| 部位  | 类型   | 血量   |
| :---- | :----- | :----- |
| 部位1 | 软组织 | 12,000 |
| 部位2 | 软组织 | 12,000 |
| 部位3 | 弱点   | 11,000 |

常规Boss，7部位，总血量500,000：

| 部位  | 类型   | 血量        |
| :---- | :----- | :---------- |
| 部位1 | 软组织 | 100,000     |
| 部位2 | 软组织 | 90,000      |
| 部位3 | 重甲   | 120,000     |
| 部位4 | 重甲   | 100,000     |
| 部位5 | 弱点   | 50,000      |
| 部位6 | 软组织 | 40,000      |
| 部位7 | 弱点   | 0（已分完） |

### 5.3 部位血量占比说明

总血量按部位数量分配，每个部位独立计算。部位总数超过5时，最后几个部位可能分不到血量，此时部位数量自动缩减。

## 六、部位护甲

| Boss阶段  | 软组织护甲 | 重甲护甲 | 弱点护甲 |
| :-------- | :--------- | :------- | :------- |
| 新手Boss  | 0          | 5        | 0        |
| 常规Boss  | 5          | 30       | 5        |
| 高阶Boss  | 15         | 80       | 10       |
| Raid Boss | 40         | 250      | 25       |
| 终极Boss  | 100        | 800      | 60       |

新手Boss软组织护甲为0，弱点护甲为0，重甲护甲为5。新手总攻击力约38，打重甲每击伤害=38×0.4-5=10。新手Boss重甲部位不建议配置，或仅配置1个作为破甲流引导。

## 七、Boss循环规则

### 7.1 Boss池

管理员在后台配置一个Boss列表，每个Boss有独立的阶段、部位布局、属性。

### 7.2 循环方式

管理员从Boss池中选择若干个Boss组成循环队列。Boss死亡后自动切换到队列中的下一个。

示例队列：

text

```
新手木桩 → 史莱姆王 → 骷髅将军 → 暗影法师 → 炎魔 → 冰霜巨龙 → 虚空之主
```



### 7.3 循环规则

| 规则     | 说明                                         |
| :------- | :------------------------------------------- |
| 顺序推进 | 按队列顺序逐个出现，不可跳跃                 |
| 循环     | 队列最后一个Boss死亡后，回到队列第一个       |
| 动态调整 | 管理员修改队列后，当前Boss死亡时按新队列执行 |
| 全服同步 | Boss切换后所有玩家立即看到新Boss             |

## 九、数值验证

新手Boss基础血量35,000，单人手动，总攻击力38：

| 部位  | 类型   | 血量   | 护甲 | 每击伤害 | 所需点击 | 耗时(5~8次/秒) |
| :---- | :----- | :----- | :--- | :------- | :------- | :------------- |
| 部位1 | 软组织 | 12,000 | 0    | 38       | 316      | 40~63秒        |
| 部位2 | 软组织 | 12,000 | 0    | 38       | 316      | 40~63秒        |
| 部位3 | 弱点   | 11,000 | 0    | 95       | 116      | 15~23秒        |

总计748次点击，约1分35秒~2分30秒。符合1~3分钟目标。

如果其中一个部位改为重甲（护甲5）：每击伤害=10，血量12,000需要1,200次点击，单部位就需要2分30秒~4分钟。建议新手Boss不配置重甲部位，或重甲护甲设为0。

## 七、Boss循环规则

### 7.1 Boss池

管理员在后台配置一个Boss列表，每个Boss有独立的阶段、部位布局、属性。

### 7.2 循环方式

管理员从Boss池中选择若干个Boss组成循环队列。Boss死亡后自动切换到队列中的下一个。

示例队列：

text

```
新手木桩 → 史莱姆王 → 骷髅将军 → 暗影法师 → 炎魔 → 冰霜巨龙 → 虚空之主
```



### 7.3 循环规则

| 规则     | 说明                                         |
| :------- | :------------------------------------------- |
| 顺序推进 | 按队列顺序逐个出现，不可跳跃                 |
| 循环     | 队列最后一个Boss死亡后，回到队列第一个       |
| 动态调整 | 管理员修改队列后，当前Boss死亡时按新队列执行 |
| 全服同步 | Boss切换后所有玩家立即看到新Boss             |

## 八、Boss配置字段汇总

| 字段        | 类型   | 说明                                           |
| :---------- | :----- | :--------------------------------------------- |
| bossId      | string | Boss唯一标识                                   |
| bossName    | string | Boss名称                                       |
| stage       | string | 阶段：新手/常规/高阶/Raid/终极                 |
| baseTotalHP | int    | 基础总血量（系数1.0时）                        |
| parts       | array  | 部位列表，每个部位含坐标、类型、血量占比、护甲 |
| goldOnKill  | int    | 手动击杀奖励金币                               |
| stoneOnKill | int    | 手动击杀奖励强化石                             |

## 九、数值验证

新手Boss基础血量35,000，单人手动，总攻击力38：

| 部位  | 类型   | 血量   | 护甲 | 每击伤害 | 所需点击 | 耗时(5~8次/秒) |
| :---- | :----- | :----- | :--- | :------- | :------- | :------------- |
| 部位1 | 软组织 | 12,000 | 0    | 38       | 316      | 40~63秒        |
| 部位2 | 软组织 | 12,000 | 0    | 38       | 316      | 40~63秒        |
| 部位3 | 弱点   | 11,000 | 0    | 95       | 116      | 15~23秒        |

总计748次点击，约1分35秒~2分30秒。符合1~3分钟目标。

百人团新手Boss，系数50.0，总血量1,750,000：

| 部位  | 类型   | 血量    | 每击伤害 | 每人所需点击 | 百人团耗时 |
| :---- | :----- | :------ | :------- | :----------- | :--------- |
| 部位1 | 软组织 | 600,000 | 38       | 158          | 3~4秒      |
| 部位2 | 软组织 | 600,000 | 38       | 158          | 3~4秒      |
| 部位3 | 弱点   | 550,000 | 95       | 58           | 1~2秒      |

百人团总计约374次点击每人，总耗时7~10秒。Boss不会瞬间蒸发。

## 十、扩展预留

| 扩展方向     | 预留方式                                      |
| :----------- | :-------------------------------------------- |
| Boss特殊机制 | parts数组预留mechanic字段（再生、护盾、联动） |
| 人数系数调整 | 系数表预留后台配置入口                        |
| 限时活动Boss | Boss池支持临时插入，活动结束后自动移除        |
| Boss难度选择 | 同一Boss可配多套属性，管理员按需选用          |
| 全服伤害统计 | 记录每个Boss的总伤害贡献，预留排行榜接口      |
