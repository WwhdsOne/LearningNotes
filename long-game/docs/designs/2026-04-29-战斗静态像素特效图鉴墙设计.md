````md
# 任务：重做战斗静态像素特效图鉴墙 Demo

## 背景

当前 Demo 已经有 `battle-static-fx-demo-page`、`battle-static-fx-demo__gallery`、`fx-layer--l1/l2/l3` 等结构雏形，但仍存在方向错误：部分特效仍使用动态 Canvas、动画、渐变、glow 或“VFX截图式”表现，不符合“静态像素图鉴墙”的目标。当前页面里甚至注明了 collapse_trigger 保留动态 Canvas，这和本次验收标准冲突，需要改掉。:contentReference[oaicite:0]{index=0}

现有战斗页也已经有 `.talent-effect-overlay`、`talent-hammer-slam`、`talent-blood-fall` 等动画式覆盖层，但本任务不要改这些真实触发链路，只做内部 Demo。:contentReference[oaicite:1]{index=1}

## 目标

新增或重构一个内部 Demo 页面：`BattleStaticFxDemo.vue`

页面只用于评审静态像素特效方向，不接后端，不改 BattlePage，不使用 OSS 特效图，不播放动画。

## 强制原则

1. 不使用 OSS 图片。
2. 不使用 Canvas。
3. 不使用 GIF。
4. 不使用序列帧。
5. 不使用 CSS animation。
6. 不使用 blur、filter、box-shadow glow、渐变光效。
7. 所有效果必须是静态像素 sprite 风格。
8. 可以用纯 CSS `box-shadow` 像素点阵画，也可以用内联 SVG，但必须保持硬边像素风。
9. 所有效果容器统一加：
   - `image-rendering: pixelated;`
   - `pointer-events: none;`
10. L4 信息层永远最高，名称、血条、数值不得被遮挡。

## 页面入口

在前端本地路由判断里新增隐藏入口：

```ts

  return BattleStaticFxDemo
}
````

不要加入公开导航。

新增：

```txt

```

如项目没有集中路由，就在当前 `App.vue` 或页面分支判断中接入。

## 层级 z-index

```css
.fx-layer--l1 { z-index: 1; }
.fx-layer--l2 { z-index: 2; }

```

L3 允许 `overflow: visible`，Boss 格子本身也要允许外扩：

```css
.battle-static-fx-demo__boss-cell {
  position: relative;
  overflow: visible;
}
```

## 静态像素实现方式

所有特效用 `<span class="fx-static fx-static--xxx"></span>`。

禁止写：

```css
animation
transition
filter: blur
box-shadow: 0 0
linear-gradient
radial-gradient
opacity 动画
canvas
img src="http"
img src="oss"
```

允许用 `box-shadow` 画像素点阵，例如：

```css
.fx-static--bleed::before {
  content: '';
  position: absolute;
  width: 4px;
  height: 4px;
  background: #7f1d1d;
  box-shadow:
    8px 0 #991b1b,
    12px 4px #7f1d1d,
    4px 8px #450a0a,
    18px 10px #991b1b;
}
```

## 每个特效的 CSS 方向

### storm_combo

绿色斜向像素斩击，L3。

要求：

* 只用 6 到 12 个硬边像素块。
* 从左下到右上。
* 不要连续光带。
* 不要 glow。

类名：

```css
.fx-static--storm-combo
```

### auto_strike

两部分：

```html
<span class="fx-static fx-static--auto-strike-hammer"></span>
<span class="fx-static fx-static--auto-strike-dent"></span>
```

* hammer 放 L3
* dent 放 L1

表现：

* 锤体只画粗轮廓，不要高清武器。
* 落点用灰色压痕和碎片。

### bleed

L1。

表现：

* 暗红色血迹像素群。
* 不要血滴，不要下落，不要液体渐变。

### omen_harvest

L3。

表现：

* 外缘镰刀弧线。
* 用离散紫黑/暗绿像素块组成，不要连续曲线。

### final_cut

L3。

表现：

* 比 storm_combo 更细。
* 深红/暗紫色。
* 中间 1 到 2 个高亮像素即可。

### collapse_trigger

必须改成全静态，禁止 Canvas。

三部分：

```html
<span class="fx-static fx-static--collapse-crack"></span>   <!-- L1 -->
<span class="fx-static fx-static--collapse-shield"></span>  <!-- L2 -->
<span class="fx-static fx-static--collapse-burst"></span>   <!-- L3 -->
```

表现：

* L1：格子内裂纹
* L2：破盾主体
* L3：外扩碎片
* 中心不要被填满

### judgment_day

两部分：

```html
<span class="fx-static fx-static--judgment-crack"></span>  <!-- L1 -->
<span class="fx-static fx-static--judgment-impact"></span> <!-- L3 -->
```

表现：

* 裂地像素线。
* 外扩冲击片。

### doom_judgment

L2。

表现：

* 断裂像素环。
* 环不能闭合成光滑圆。
* 不要魔法阵文字，不要符文细节。

### silver_storm

L3。

表现：

* 3 到 4 段银色短斩击。
* 分散在格子边缘。
* 不要雾化风暴。

### crosshair-mark

L2。

表现：

* 中心准星。
* 四个方向短线。
* 线条必须硬边像素。
* 不要现代 UI 光效。

不要修改：


## 验收标准

完成后检查：

1. `/internal/battle-static-fx-demo` 可以打开。
2. 页面有 10 张单特效卡片。
3. 页面底部有 3 张并发组合卡片。
4. 页面没有 `<canvas>`。
5. 页面没有 OSS 图片 URL。
6. 页面没有 CSS animation。
7. 所有效果都是静态像素构图。
8. 名称、血条、血量、护甲信息始终在最上层。
9. collapse_trigger 不再使用动态 Canvas。
10. 多特效组合时主体仍然可读。

## 重要修正

不要再按“特效动画截图”做图。

本 Demo 的正确方向是：

> 静态像素 UI 贴图，不是 VFX。

所以所有命名、CSS、结构都要围绕：

* 点阵
* 硬边
* 少量颜色
* 可叠加
* 不遮挡信息

```

核心一句：**让 Claude Code 先做纯 CSS 点阵版，不要再碰 AI 生成图。**  
等图鉴墙构图确认后，再把每个 CSS 点阵替换成真正手修 PNG。
```
