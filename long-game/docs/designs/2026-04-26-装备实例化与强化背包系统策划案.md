## 一、动机与目标

### 1.1 现状问题

- 当前背包使用 `HASH itemId → quantity` 存储，强化只能跟 itemId（原型）走，无法个体追踪
- 后续需要交易系统，每个装备必须是可独立标识的实例
- 背包为垂直列表展示，装备栏和背包分属不同 Tab，信息密度低

### 1.2 设计目标

1. **实例化**：每件装备的每个拷贝是一个独立 `instanceId`，各自携带 `enhanceLevel`
2. **WoW 风格背包**：方格展示、悬浮提示、右键菜单
3. **页面整合**：装备栏 + 属性预览 + 背包合并为单一个人页
4. **强化系统**：指数消耗 + 指数成长，100% 成功
5. **装备不堆叠**：每个实例独立占一个背包格
6. **背包无上限**

## 二、实例化物品系统

### 2.1 核心概念

- **原型（Prototype）**：`vote:equip:def:<itemId>` HASH，描述装备种类（名称、部位、稀有度、基础属性等）。由管理后台创建。
- **实例（Instance）**：`vote:instance:<instanceId>` HASH，描述一个具体的装备拷贝（itemId、enhanceLevel、spentStones 等）。由掉落系统创建。
- 原型 → 实例 = 1:N 关系。

### 2.2 实例数据

```
vote:instance:<instanceId>  HASH
    itemId: string          (引用原型)
    enhanceLevel: int       (0 = 未强化)
    spentStones: int64      (已消耗强化石累计，供分解返还)
    bound: bool             (锁定状态，预留交易系统)
    createdAt: int64        (掉落时间)
```

### 2.3 玩家实例索引

```
vote:player-instances:<nickname>  SET
    [instanceId1, instanceId2, ...]
```

- 玩家拥有哪些实例的全量索引
- 背包展示 = 遍历该 SET 的每个 instanceId，读取实例数据 + 原型数据
- 每个实例独立生成一个 `InventoryItem`，quantity 恒为 1

### 2.4 装备栏改造

当前：`vote:user-loadout:<nickname>` HASH `slot → itemId`
改为：`vote:user-loadout:<nickname>` HASH `slot → instanceId`

- 穿戴时把实例 ID 写入对应槽位
- 卸下时删除槽位
- 战斗统计读取 loadout 中各 instanceId 的 enhanceLevel，计算增强属性

### 2.5 旧数据结构

```vote:user-inventory:<nickname>``` HASH `itemId → quantity` 保留不动，不进行数据迁移。老玩家的旧数据不动，新掉落全部走实例化通道。

## 三、背包系统

### 3.1 展示方式

- CSS Grid 方格布局，自动列数（`repeat(auto-fill, minmax(72px, 1fr))`）
- 每格 72x72px 正方形
- 有装备的格子：显示装备图标（imagePath），无图标则用稀有度颜色填充方块
- 空格子：半透明虚线边框
- `overflow-y: auto` 滚动，背包不设大小上限

### 3.2 格子视觉

| 状态 | 表现 |
|------|------|
| 空格 | 浅色虚线边框，透明度 0.3 |
| 有装备 | 稀有度颜色边框 + 图标/色块 |
| 已装备 | 额外金色边框或角标 |

稀有度颜色：
| 稀有度 | 颜色 |
|--------|------|
| 普通 | #6b7280 |
| 稀有 | #2563eb |
| 史诗 | #c026d3 |
| 传说 | #d97706 |
| 神话 | #ef4444 |
| 至臻 | 彩虹渐变 |

### 3.3 交互

| 操作 | 行为 |
|------|------|
| 鼠标悬停 | 弹出 Tooltip 浮层 |
| 左键单击 | 未装备 → 装备，已装备 → 卸下 |
| 右键单击 | 弹出上下文菜单 |

## 四、装备强化

### 4.1 基础规则

- 强化 100% 成功
- 强化只增加 `enhanceLevel`
- 强化上限由稀有度决定
- 当前版本无突破系统（UI 可预留锁定态）

### 4.2 上限

| 稀有度 | 基础上限 |
|--------|---------|
| 普通 | 5 |
| 稀有 | 10 |
| 史诗 | 15 |
| 传说 | 20 |
| 神话 | 25 |
| 至臻 | 30 |

### 4.3 消耗公式

```
本级金币消耗     = 100 × 1.5^currentLevel
本级强化石消耗   = 1 × 1.4^currentLevel
```

### 4.4 属性增长公式

```
数值类属性    当前值 = baseValue × 1.12^enhanceLevel
百分比类属性  当前值 = baseValue × 1.08^enhanceLevel
真伤比例类    当前值 = baseValue × 1.02^enhanceLevel（每 +3 级展示一次增长）
特殊机制类    不随强化等级成长
```

### 4.5 强化流程

1. 右键菜单选择"强化"
2. 弹窗展示：当前 +N → +N+1，消耗金币/强化石
3. 资源不足时消耗项红字提示
4. 确认后调用 `POST /api/equipment/:itemId/enhance`
5. 后端：找一张未装备的实例 → 扣资源 → 提升等级 → 刷新状态

### 4.6 强化视觉

| 等级 | 表现 |
|------|------|
| +0~+4 | 无特殊 |
| +5~+9 | 边框微光 |
| +10~+14 | 细粒子飘散 |
| +15~+19 | 呼吸光效 |
| +20~+29 | 强光晕和粒子环绕 |
| +30~+59 | 彩虹光晕和多层光环 |
| +60~+99 | 中心光柱和高速光环 |
| +100 | 全屏彩虹爆发后稳定专属光环 |

## 七、数据模型

### 7.1 Redis Key 一览

| Key 模式 | 类型 | 用途 |
|----------|------|------|
| `vote:equip:def:<itemId>` | HASH | 装备原型定义（已有） |
| `vote:equipment:index` | SET | 所有原型 ID 索引（已有） |
| `vote:instance:<instanceId>` | HASH | 装备实例数据（新增） |
| `vote:player-instances:<nickname>` | SET | 玩家拥有的实例 ID 集合（新增） |
| `vote:user-loadout:<nickname>` | HASH | slot→instanceId（改造） |
| `vote:resource:<nickname>` | HASH | 金币/强化石/宝石（已有） |
| `vote:user-equipment-spent:<nickname>` | HASH | 旧消耗记录（改造为 instance 级） |
| `vote:user-inventory:<nickname>` | HASH | 旧 itemId→quantity（保留不动） |

### 7.2 API 接口

**新增：**
| 方法 | 路径 | 用途 |
|------|------|------|
| POST | `/api/equipment/:itemId/enhance` | 强化一件装备 |
| POST | `/api/equipment/:itemId/discard` | 丢弃一件装备 |

**现有（改造为 instance 感知）：**
| 方法 | 路径 | 改造点 |
|------|------|--------|
| POST | `/api/equipment/:itemId/equip` | loadout 存 instanceId |
| POST | `/api/equipment/:itemId/unequip` | 按 instanceId 找 slot |
| POST | `/api/equipment/:itemId/salvage` | 基于实例读 spentStones |

**现有（无需改动）：**
| GET | `/api/player/profile` | 返回 InventoryItem[]（已含实例数据） |

### 7.3 InventoryItem 扩展

```go
type InventoryItem struct {
    ItemID               string  `json:"itemId"`
    Name                 string  `json:"name"`
    Slot                 string  `json:"slot"`
    Rarity               string  `json:"rarity"`
    ImagePath            string  `json:"imagePath,omitempty"`
    ImageAlt             string  `json:"imageAlt,omitempty"`
    Quantity             int64   `json:"quantity"`           // 恒为 1
    Equipped             bool    `json:"equipped"`
    AttackPower          int64   `json:"attackPower,omitempty"`
    ArmorPenPercent      float64 `json:"armorPenPercent,omitempty"`
    CritRate             float64 `json:"critRate,omitempty"`
    CritDamageMultiplier float64 `json:"critDamageMultiplier,omitempty"`
    PartTypeDamageSoft   float64 `json:"partTypeDamageSoft,omitempty"`
    PartTypeDamageHeavy  float64 `json:"partTypeDamageHeavy,omitempty"`
    PartTypeDamageWeak   float64 `json:"partTypeDamageWeak,omitempty"`
    // 新增实例字段
    InstanceId           string  `json:"instanceId"`
    EnhanceLevel         int     `json:"enhanceLevel"`
    Bound                bool    `json:"bound"`
}
```
