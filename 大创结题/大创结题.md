# 作品名称 ：智能对话与知识库管理平台

# 功能介绍

技术架构：采用前后端分离架构。后端：Go + GORM + PostgreSQL + Redis + Consul + OpenAI API；前

端：Vue 3 + Element-Plus；部署：Nginx + GitHub Actions CI/CD。后端采用三层架构（接口层 → Service

层 → Repository 层 → DB/Redis），结合 Wire 进行依赖注入管理。

后端核心开发：

• 数据库优化：设计基于 PostgreSQL 的业务模型，改造 GORM 主键生成策略，解决跨环境迁移一

致性问题。

• 缓存与工具链：自研 Redis 封装工具，实现缓存读写、过期策略及异常兜底；基于滑动窗口算法

实现高并发请求限流，并统计热 key 访问频率，为业务决策与性能优化提供数据支撑。

• 对话与问答系统：开发会话与消息管理接口，支持对话标题自动总结；基于 RAG 架构实现知识库

问答，结合向量检索与精排（Rerank）模型提升答案相关性与准确度。

• 中间件与日志：基于 Gin 框架编写日志中间件，实现请求链路日志、性能指标采集及错误监控；

编写统一异常处理中间件，实现验证错误、数据库错误及通用错误统一捕获与响应。

• 配置与部署：接入 Consul 实现动态配置管理；构建自动化 CI/CD 流水线，完成服务的自动构建与

部署；修改 Nginx 配置，优化路由，并集成 Certbot 自动化 SSL 证书更新，实现 HTTPS 自动续期。

• 性能优化：升级服务端至 Go 1.25，利用 Green Tea GC 和 Go profiling 工具优化内存使用和垃圾回

收效率，降低 P99 响应延迟约 15

前端全功能开发：

• 核心功能模块：独立开发知识库管理、对话管理及模型配置页面，实现核心功能交互逻辑并优化

操作体验。

• 聊天交互体验：实现历史会话的创建与读取；优化聊天输入框（支持 Enter 发送、动态扩容、最大

高度限制），极大提升用户输入体验。

• 内容渲染优化：深度定制 Markdown 渲染组件，实现流式（实时）渲染 AI 返回内容，支持代码块

高亮、行号、复制按钮及数学公式渲染，提升用户交互体验。

• UI/UX 细节：修复 Element-Plus 组件库样式问题（如选择框错位），统一模型选择图标格式，确保

界面美观与交互一致性。
